### Post #1 — @Martin-mx (2020-05-31)

This bugfix release is fixing a problem appeared in 3.2.5 with possibility of duplicate inputs for transaction assembler, see [#1126](https://github.com/ergoplatform/ergo/issues/1126) for details.

Other changes in the code are mostly cosmetic, aside of changing values for replace-compact-collect box selector (they still hardcoded, see [#856](https://github.com/ergoplatform/ergo/issues/856) ).
