### Post #1 — @kushti (2025-04-09)

* Ergo protocol reference client 6.0.0-RC4 released! [Release Ergo protocol reference client 6.0.0-RC4 · ergoplatform/ergo · GitHub](https://github.com/ergoplatform/ergo/releases/tag/v6.0.0-RC4), now can be used in the mainnet (but voting details not set, and there will be some changes towards final 6.0 release!)
* Developers, check Ergo Basic Template [GitHub - ERGnomes/ergo-basic-template: A basic site for connecting with the ergo platform](https://github.com/ERGnomes/ergo-basic-template) , a modular React template for building applications on the Ergo blockchain. This template provides a structured foundation with reusable components for wallet connection, token display, and a beautiful dashboard interface.
* A new Layer 2 scheme proposed, where participants can freely create money (transferable IOU notes), but redemption is limited: [Offchain money with free creation but limited redemption](https://www.ergoforum.org/t/offchain-money-with-free-creation-but-limited-redemption/5122)
* A new BtcFi scheme published, where Ergo contracts can be used to insure a Bitcoin on-ramp [Insuring Bitcoin on-ramp with Ergo contracts](https://www.ergoforum.org/t/insuring-bitcoin-on-ramp-with-ergo-contracts/5121)
* Mew Finance launched a seamless 2 way cross-chain NFT bridge connecting the Cardano and Ergo ecosystems. <https://Bridge.mewfinance.com>
* Mew Finance published Q1 report [Mew Finance Quarterly Business Update: Q1 2025 | by Mew Finance | Apr, 2025 | Medium](https://medium.com/@themewfinance/mew-finance-quarterly-business-update-q1-2025-f864bb9b9454)
* Check the new article " The Money Game: Rewriting the Rules with Cryptocurrency" by Joe Armeanio from the upcoming ErgoBook [ergo-book/Ideological Background/The Money Game: Rewriting the Rules with Cryptocurrency at master · kushti/ergo-book · GitHub](https://github.com/kushti/ergo-book/blob/master/Ideological%20Background/The%20Money%20Game%3A%20Rewriting%20the%20Rules%20with%20Cryptocurrency)
* ChainCash: updated paper is done and can be found @ [chaincash/docs/conf/conf.pdf at master · ChainCashLabs/chaincash · GitHub](https://github.com/ChainCashLabs/chaincash/blob/master/docs/conf/conf.pdf)
* Ergo Vanity Address Generator updated [GitHub - arkadianet/ergo-vanitygen-rust: A high-performance vanity address generator for Ergo blockchain](https://github.com/arkadianet/ergo-vanitygen-rust)
* The keystone team started reviewing pgr456’s PRs. 2 out of 3 are already merged. Currently working together with them on reducing firmware size and other requested changes (see discussion with dusek here [Discord](https://discord.com/channels/668903786361651200/669989266478202917/1356224220530737192)). According to them Ergo will be included in the cypherpunk version of the keystone firmware (bitcoin, zcash, monero and ergo). Current progress moved here: [Ergo by soralit · Pull Request #1676 · KeystoneHQ/keystone3-firmware · GitHub](https://github.com/KeystoneHQ/keystone3-firmware/pull/1676)
* SkyHarbor v2 frontend now open source: [GitHub - skyharbor-market/frontend-skyharbor-ts](https://github.com/skyharbor-market/frontend-skyharbor-ts)
* The (hopefully) last change requested by Ledger before the official release is merged ([Add blind signing warning by arobsn · Pull Request #12 · LedgerHQ/app-ergo · GitHub](https://github.com/LedgerHQ/app-ergo/pull/12#event-17186403649)).
