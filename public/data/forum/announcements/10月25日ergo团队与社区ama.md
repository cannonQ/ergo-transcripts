**问题2：**

亲爱的Ergo团队，你们有没有计划聘请专业的传播人员或营销专业人士来向大众推广Ergo？我想我可以代表这里的大多数持有者，我们真的希望看到Ergo在大众面前拥有更高的知名度，更能家喻户晓。因为尽管该项目提供了强大的技术和价值，但仍然相对默默无闻。感谢您的回复！谢谢。

Glasgow：

Ergo基金会一旦成立，便会推出扩张计划。但是这些流程需要花费大量的时间。现在几乎所有工作都由我们完成（漫长的身份认证流程和大量的表格）。

我们在幕后的工作中已经有品牌重塑的计划，这将与Ergo基金会成立和新的Ergo基金会标志联系起来。其目的是将重心带回到“第一原则”和草根去中心化金融。我们会明确区分Ergo基金会和社区。积极的一面是——一家广告设计公司伸出了援手（事实证明他们有许多人都是矿工），我们正在制作一个专业的广告，可以传达信息并简化一些概念。（译注：“第一原则”是由亚里士多德提出的哲学与逻辑概念，指最基本的命题或假设，不能省略、删除或违反。Ergo宣言中明确规范了关于去中心化、安全性、社会公益等五项Ergo必须遵守的基本原则，即为Ergo的“第一原则”）

编辑团队还在撰写《Ergo与区块链：加密货币领域》系列文章，“我们想在继续推动主流采用的同时，揭开加密货币许多概念和原则的神秘面纱，并将它们简化。”

值得注意的是，直到最近，我们的发展都非常好。今年年初，我们的市值排在400名左右，Reddit相关论坛的浏览量在六个月内从0增加到100万。这段时间的营销从未停歇，但专注于工程发展范围内的结构性内容。改善客户体验、沟通途径、收集和构建资源、整合开发文档并确保工具对于普通人确实是可访问和可用的。

我们接纳了大量新开发者（有偿或无偿），并且在短短几个月内已经取得了长足的进步。由于精妙的脚本语言和各种可开发的应用程序，开发者们被Ergo吸引。随着Ergo Pay、Yoroi dApp连接器和其他重要改进在未来几个月出现，它对普通人来说将变得更加可用。

我们已经看到了大量矿工涌入——当利润更高时，矿工们将不断涌入，并接触到现在发布的dApp和工具。不幸的是，该算法在指数增长下的表现要好于比特币盘整引起的汹涌浪潮。

Armeanio一直在以惊人的速度添加矿池、像智能矿池（smart-pool）和Subpooling（先决条件）这样的应用，并开始形成规模。

ETH PoS已经推到了5月或7月，因此我们将在这段时间内积累更多。

就个人而言，我很兴奋，因为要么GPU会变得非常便宜，要么一直计算挖矿如何结束（从不考虑价格）的矿工们错了。

Armeanio：

关于ETH PoW结束后的矿工迁移有很多未知数。

结束日期本身已更改多次…

当我刚着手这项工作时，这是我的首要任务，因为我们已经让Nanopool处于算力主导地位。

我认为我们已经增加了足够的多样性和竞争，从而应该能够吸收算力的大幅增长，而不会再次发生这种情况（单一矿池算力占比过半）。

我认为这对Ergo来说是一个巨大的机会，因为我认为大多数用GPU挖矿的PoW区块链没有太多的长期可行性。大多数只是缺乏承载大型复杂生态系统所需的工具。

**问题5：**

有没有在新交易所上币的计划？你们在朝这个方向努力吗？谢谢。

Armeanio：

我已经向每个当前靠谱可行的交易所（不要求成立实体）都提交了上币申请，这些申请都已更新并记录在案。

在大多数情况下，会有代码编写和法律审查/审计的过程。然后是整合我们的主网，这需要定制方案来整合一条新链。之后，上币一旦获得批准，通常会排队等待。

在牛市中，排队时间会变长，因为有很多新代币获得风投资金并试图购买市场准入，尽管在许多情况下它们的核心非常空洞。

许多加密货币资产都不是原生区块链，而是构建在另一个网络之上的原生代币。由于不需要实际的工作，它们可以快速添加到交易所。

许多其他区块链是分叉的，或者至少具有与现有代码库非常相似的属性。

Ergo与众不同，它很美，但也会拖慢整合过程的脚步。

我的长期目标是让ERG在每个主要交易所上币。

我们的市值仍然较低，但随着我们实现和灵活运用Ergo网络的功能和工具，我认为与大多数其他区块链的技术能力相比，我们处于有利地位。使用和实用性是长期价值的基础。

完成Ergo基金会成立流程将有助于Ergo基金会能够在多个司法管辖区开展业务。

Glasgow：

Armeanio正在与所有交易所沟通，他们都很慢。

随着ETH网关的完成，Graviton使我们能够在未来几周内很快用上DEX。

**问题** **6**

可扩展性的计划是什么？是否有计划实现类似于闪电网络的东西？

有没有办法从加密货币挖矿中众包算力来解决现实世界中的问题，比如找到治愈疾病的方法？（类似于folding@home项目）

谢谢你们的努力工作！我迫不及待地想看看Ergo平台将如何向前发展。

Kushti：

闪电网络可以建立在Ergo之上（我们在2018年对此进行了研究），因此我猜可以将某些比特币闪电网络客户端重新定位到Ergo（就像比特币分叉所做的那样）。然而，即使是比特币，闪电网络的采用也很慢，所以很难说什么时候部署比较好。

Ergo允许更有吸引力的Layer 2解决方案，例如只需支付很少的费用即可运行LETS系统（本地交易所交易系统）。

关于有用的PoW，有一些阐述了候选方案的论文，但这些初期研究在现实世界中通常是有问题的。

Armeanio：

可扩展性是UTXO联盟的主要目标之一。

### [UTXO Alliance](https://utxo-alliance.org)

Blockchain-focused, asset-agnostic initiative, created to kickstart collaboration across blockchains with shared characteristics.

我们正在对当前的可扩展解决方案进行研究。Ergo就是考虑到这一点而构建的，可以支持各种预先存在的解决方案。我们可能会看到我们的联合研究伙伴开发了新框架，但现在还为时过早。

以下是来自论坛的Kushti可扩展性帖子摘要。帖子刚刚更新了Ergo协议研究和客户端开发路线图。

Ergo针对交易拥堵的解决方案如下：

L0：从v4.0.8版开始，节点已经完成了大量的增效改进，20-50倍的增益在这里仍然是可能的。我们还计划使用NiPoPoW证明和UTXO集合快照进行快速引导

L1：Ergo在其代码中有一个扩展部分，它允许实现各种扩展解决方案，例如Sharding（分片）、Hydra或BitcoinNG风格的微区块。这甚至可以让我们使用丝绒分叉或软分叉来做通用侧链。

L2（链下）——Ergo应该与闪电网络、彩虹网络等很多网络兼容。此处的实现将取决于在Ergo上构建的应用程序之需求。

无状态客户端、NIPoPoW、树莓派上的全节点、超高效SPV客户端以及其他即使在负载下也能长期工作的方法。存储租赁费可以防止垃圾攻击和粉尘（极微小量的加密货币），并且稳定挖矿收入。

收取存储租赁费的其他好处包括防止“状态膨胀”、围绕状态建立经济（用户必须付费才能将未使用的盒子长期保存在矿工的内存中）以及逐步找回任何丢失的币，并让其重新进入流通。

**问题11：**

1.您能描述一下您关于挖矿基础设施的理想或目标吗？随着子池化（Subpooling）和对数挖矿的到来，挖矿是否正在从完备的专业矿机转向其他形式？例如，它是否可以在移动设备、浏览器或其他不涉及显卡和挖矿软件的轻客户端上实现？

2.关于改变挖矿和发行计划的讨论是否也会激励Ergo实现问题1中的愿景？

Kushti：

1.我认为显卡挖矿很好，CPU挖矿可能会出现僵尸网络问题，移动设备挖矿看似美好但是前景尚不明朗。

2.发行时间表更多的是关于挖矿奖励的稳定性，因此网络稳定性应该是可预测的。目前对于改变挖矿没什么好说的。

3.比特币的Taproot升级激活后，我们可以进行实验性隐私交换。然而，我们仍然没有足够好的原子交换DEX，因此很难预测基于隐私交换的DEX能否成功。也许除了交换之外的有趣应用会出现在Schnorr上，否则，隐私交换不会为比特币带来新的合约。带有比特币的侧链可以为比特币带来合约（例如非交互式混合）。

**问题14：**

您认为在发行结束后，存储租赁费是否足以吸引和留住矿工？

Glasgow：

请参阅官方论坛有关通过软分叉延长发行的帖子。

我认为需要担心的不是它的发行量会不足——而是你绝对需要有足够的量（而且很难计算）。

简单来说：取今天发行的ERG一部分，并将其与额外的交易费用（以及在发行结束前累积的存储租赁费）一起放入合约中，作为协议稳定性的纠错保障。

MHS\_Sam：

每个交易（或至少所做的状态更改）都需要在每个节点存储，因此如果人们进行数千笔交易，每个节点都需要存储更多信息。这导致中心化，因为节点运营者更难同步，空间需求开始成为一个问题。

L2解决方案是正确的一步。这不仅仅是关于吞吐量，而是TPS、安全性和去中心化之间的平衡。如果保护或参与网络所需的硬件超出了普通用户的范围，我认为这不是P2P货币的可行机制。

如果您关闭一些数据中心，结果区块链出现了故障，那么它的适应力就没有那么强了。如果它没有适应性，我们就称之为私有数字货币，而不是加密货币。有一些L1链挺不错的，但总有一些取舍。

ErgoMixer是一个建立在Ergo区块链之上的dApp。

开发者不是Ergo核心团队的一部分。该dApp的存在是由于Ergo和Sigma协议的可编程性。很多交易所问过我这个问题，我通常回答说，拥有高度可编程的区块链将催生各种dApp。这就是可编程性的本质，人们会构建不同的应用程序，用户会选择使用什么。最终，用户需要为其承担责任和承受风险。

Ergo’s solution for transaction congestion is as follows:

L0: Starting from the v4.0.8 version, the node has completed a large number of synergy improvements, and a gain of 20-50 times is still possible here. We also plan to use NiPoPoW proof and UTXO collection snapshots for quick boot

L1: Ergo has an extension in its code, which allows various extension solutions to be implemented, such as Sharding (sharding), Hydra or BitcoinNG style micro-blocks. This even allows us to use velvet forks or soft forks to make general-purpose side chains.

L2 (off-chain)-Ergo should be compatible with many networks such as Lightning Network and Rainbow Network. The implementation here will depend on the needs of the application built on Ergo.
