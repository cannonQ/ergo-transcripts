# Fleet SDK v0.1.3降低了Ergo dApp开发门槛

开发者是任何加密货币生态系统的支柱，他们负责创建我们每天使用的许多 dApp（去中心化应用）。一般来说，区块链能否蓬勃发展不仅与生态系统中的开发者人数呈正相关，还与在区块链上开发的难易程度呈正相关。在某些区块链上开发的问题是只有某些编码语言兼容。然而，由于著名的[Ergo开发者Captain Nemo](https://twitter.com/NautilusWallet)（[Nautilus钱包](https://chrome.google.com/webstore/detail/nautilus-wallet/gjlmehlldlphhljhpnlddaodbjjcchai)的创始人）最近的更新，开发变得更加容易。

**最近的更新** **有何** **作用** **且** **有** **哪些** **好处？**

Captain Nemo最近在Ergo上推出了[Fleet SDK v0.1.3](https://github.com/fleet-sdk/fleet)。他们[指出](https://twitter.com/NautilusWallet/status/1682502973115576321)，“这个版本带来了ErgoScript编译器和‘模拟链’包，让您可以使用Jest和Vitest等流行的测试框架在模拟区块链环境中测试您的Ergo合约。”

对于不懂技术的人来说，可能会问这到底意味着什么。从本质上讲，开发者现在有机会利用世界上最流行的两种编程语言JavaScript和TypeScript来构建Ergo dApp。再加上Ergo的开源性质，这降低了在Ergo上进行开发的门槛，并可以促进该协议得到更广泛的采用。想要在Ergo上构建dApp的开发者不需要从头开始，我们鼓励他们在其他开发者的代码上进行创新。

对于进入新生态系统的开发者来说，这可能会让人不知所措。查找获取某些编码语言知识所需的资源、教育信息和其他文档并不总是那么容易。然而，Ergo提供了大量的[资源](https://github.com/ergoplatform/awesome-ergo)来帮助新人了解开发。

在Captain Nemo的Fleet v0.1.3更新中，需要注意的是，SDK（软件开发工具包）充当了一个构建良好的框架，其中包含预构建的工具、代码库和资源，以利于加快编码过程。这一重要的基础设施无疑将吸引更多开发者加入Ergo生态系统。 这些节省时间的工具使设计选择成为优先事项，确保未来的dApp具有尽可能高的质量。

Ergo最大的一个优点是可以轻松联系上开发者（核心开发者和社区开发者）。具体来说，Ergo的Discord开发服务器是志同道合的开发者共享、辩论和相互交流想法以帮助解决问题的地方。

为了促进区块链的采用，开发需要向所有领域的开发者广泛开放。Ergo致力于为在Ergo上进行设计和构建的开发者创造无缝过渡。创建无需许可、开源且透明的代码是Ergo精神的一部分，Captain Nemo的这一重要更新是为普通人创造更多金融工具的一大进步。
