# Ergo General Chat — 2024-W40 Summary

## Key Topics Discussed

- **Sub-blocks naming and design**: Discussion on naming conventions (sub-blocks vs. weak blocks vs. input/contract blocks) and whether political outcomes affect implementation. kushti clarified sub-blocks are real progress with multiple naming proposals under consideration [@dayumbbbb, msg#517213-517250].

- **Block time and throughput implications**: kushti explained that increasing block speed affects timestamp-based contracts and requires careful protocol consideration; mixing transactions (8x weight vs. payments) change the TPS calculation significantly [@kushti, msg#517723-517743].

- **Transaction fee pressure during high load**: Reference to autumn 2020 when mixers created load and exchanges raised fees from 0.001 to 0.1 ERG; block limits can be raised further [@kushti, msg#517709].

- **Palmyra and Ergo capacity**: Discussion on whether Ergo can handle increased transaction load from Palmyra NFT minting; bulk minting protocols using authenticated trees proposed as optimization [@kushti, msg#517716-517718].

- **Ledger support expansion**: 4,100 ERG bounty offered for C developer to add UI support for Ledger Stax and Flex devices; new developer confirmed to start 2024-10-07 [@kushti, msg#517905, msg#517989].

- **SigmaUSD collateralization concerns**: Discussion on whether increasing treasury from 400% to 800% improves security; bridging SigmaUSD to Ethereum raises bearer asset trust assumptions vs. permissionless minting [@Armeanio, msg#517467-517472].

- **Rosen Bridge Ethereum testing**: Community testing of Ethereum integration completed; team integrating feedback before deployment [@Armeanio, msg#517604].

- **MewFinance presale results**: Presale sold out in under 3 hours with 41 participants; second presale announced [@HQΣr, msg#517210, msg#518070].

- **Satergo v1.8.0 release**: New features: UTXO consolidation, token burning, multi-address sends, airdrop from spreadsheet, Javanese translation, improved accessibility [@Aberg, msg#518219].

- **DEX performance**: MewDex swap UI significantly faster than Spectrum DEX; code cleanup by Aco contributed to improvements [@qx(), msg#518485-518488].

- **Oracle pool v2 for Dexy Gold**: Sigmanauts mining pool testing fast oracle feed distribution with GORT rewards [@Armeanio, msg#517604].

- **Ecosystem fundraising and control**: kushti proposed smarter fundraising mechanisms using Ergo contracts, including buyback guarantees and community control over spending [@kushti, msg#517753-517755].

- **Commercialization strategy**: kushti emphasized need for real-world adoption through goods/services sales in SigmaUSD/Gluon, bulk minting, p2p onramps (Mew, TokenJay), and avoiding overcollateralized stablecoin inefficiency at scale [@kushti, msg#518327, msg#518469-518471].

- **GraphQL server issues resolved**: gql.ergoplatform.com went down due to docker nginx not restarting after update; fixed 2024-10-06 [@kushti, msg#518328].

## Technical Q&A Worth Preserving

- **Q** (@Jack Σ): If Palmyra brings high transaction volume, will everyone pay more in fees? [@517708]
  **A** (@kushti): Block limits can be safely raised from current levels; mixing transactions are 8x heavier than simple payments, so TPS calculation depends on transaction mix; raspberry pis would run light clients only, while multithreading on modern CPUs can drastically improve parallel validation [@517709, 517723, 517734]

- **Q** (@Jack Σ): How will Ergo handle increased Palmyra NFT minting load? [@517703]
  **A** (@kushti): Bulk NFT minting using authenticated trees was already proposed; main limiting factor is heavy cryptographic operations, not size. Long contracts execute faster than simple payments [@517718, 517732-517735]

- **Q** (@Jack Σ): Can sub-blocks be used as fast oracle feeds for live events? [@517762]
  **A** (@kushti): Interesting idea; asked if you mean "fast oracle pools" [@517765]

- **Q** (@Jack Σ): If Uranium (DexyUra) is added to Dexy, how vulnerable would protocol be? [@518443]
  **A** (@kushti): Need multiple datasources for Uranium price; single data feed shutdown would create vulnerability. Also questioned if Uranium is too exotic for DeFi traders [@518446, 518448]

- **Q** (@Felipe): Where is the option to mark tokens as spam in ErgoExplorer? [@517967]
  **A** (@qx()): Main token list in explorer; community reports on ErgExplorer Telegram channel; threshold of 10+ unique users required to flag as scam. Takes time to sync [@517968-517976]

- **Q** (@Felipe): Will Wallet Optimization cause security risks if not performed? [@518078]
  **A** (@qx()): Risk is storage rent charges; normal wallet usage takes care of it; if worried, run optimization once every 3.9 years [@518079-518981]

- **Q** (@Daniel): Is the Ledger app safe to use while in developer mode? [@518005]
  **A** (@qx()): Yes, safe. Follow rule: never send more than 20 different tokens at once [@518011]

- **Q** (@Dmitry Usov): Who maintains the two GraphQL endpoints? [@518309]
  **A** (@qx()): PG maintains Sigma Space endpoint; qx() maintains Cornell endpoint [@518335]

- **Q** (@Jack Σ): Are transactions slower on Ergo at the moment? [@518296]
  **A** (@kushti): Asked for details [@518297, 518300]

- **Q** (@Unknown): Can you use Ledger app safely? [@517423]
  **A** (@Unknown): Yes; with Nautilus webview and Ethereum bridge, users will be able to buy ERG with USDT on Ethereum [@517423]

- **Q** (@Antonis): Is Ergo planning to be part of Cardano partnerchains? [@518064]
  **A** (@HQΣr): "Aren't we already?" [@518065]

- **Q** (@TMR.ΣRG): Why does SigmaUSD depeg when bridged to other chains? [@518483]
