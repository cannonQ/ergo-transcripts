# Ergo General Chat — 2020-W03 Summary

## Key Topics Discussed

- **Exchange liquidity and market access**: Discussion of where to acquire ERG tokens; participants noted poor liquidity on centralized exchanges and recommended Tidex and Waves.exchange trading pairs as best options for purchasing.

- **Full node synchronization**: Community member asked about sync time for a full Ergo node on 120Mbps internet; feedback indicated roughly 1 hour on adequate hardware.

- **Mining setup and configuration**: Questions about CUDA compute architecture parameters for GPU mining (RTX 2080 Ti), Java version requirements for running the Ergo node, and whether a hot wallet is mandatory for mining pool operation.

- **Waves token bridge**: Clarification requested about whether ERG tokens on Waves still need to be swapped to mainnet and how the swap mechanism works.

- **Mining pool architecture**: Discussion of distributed mining pool design, with proposal to use collateralized private keys issued by the pool to individual miners.

## Technical Q&A Worth Preserving

- **Q** (@Unknown, msg#16321): "Hi guys, I'm trying to set up a full node on Linux. How long does it take to sync and load all the blocks on a 120Mps internet?"
  **A** (@Unknown, msg#16323): "Depends on hard drive, should be around 1 hour"

- **Q** (@Unknown, msg#16328): "CUDA_COMPUTE_ARCH have 60 as default. I have a RTX 2080 Ti, should I change that parameter?"
  **A** (@Unknown, msg#16329): "Should be autodetected with the latest version of the miner"

- **Q** (@Unknown, msg#16333): "Do I need Java 9 on Windows?"
  **A** (@Unknown, msg#16335, #16338): Java 8 is the only version with official JRE support; versions above 8 are experimental. Later clarified that only Java 8 has official JRE, with versions 9-11 lacking security updates.

- **Q** (@god, msg#16390, #16391): "When there is no private key in my wallet, can I still get the task normally? Must ergo mining use a hot wallet? Is it possible to construct coinbase transactions for mining by myself?"
  **A** (@scalahub, msg#16392): "Yes as far as I know, you must have a hot wallet"

- **Q** (@god, msg#16399-#16401): Proposal for distributed mining pool design using collateralized private keys issued by the pool, asking if miners can modify or generate mining tasks themselves or if pool-generated tasks are mandatory.
  **A** (@god, msg#16409): "The miners only need to pay the collateral to the private key account address issued to them by the mining pool."
