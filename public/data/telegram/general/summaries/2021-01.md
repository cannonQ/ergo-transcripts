# Ergo General Chat — Monthly Summary: January 2021

---

## Executive Summary

January 2021 was a landmark month for Ergo, dominated by three converging themes: the Autolykos v2 / "The Hardening" protocol upgrade approaching activation, the Ergo Summit 2021 event on January 23rd unveiling the AgeUSD stablecoin architecture, and growing community and media momentum driven by the Emurgo partnership and Oracle Pools coverage. Developer and ecosystem activity accelerated significantly, with multiple DEX models, cross-chain interoperability designs, and mining infrastructure all advancing in parallel.

---

## Key Topics This Month

### Autolykos v2 and "The Hardening" Fork

The most technically urgent topic of the month was the Autolykos v2 upgrade, scheduled for block height 417,792 (approximately February 1–3). The key changes were the removal of pool resistance (after community feedback that smart contracts render it ineffective anyway, as c8 noted: no PoW chain with smart contracts can be truly pool-resistant) and a reduction of GPU memory requirements from 4GB to 2GB initially (with requirements growing over time). Reference NVIDIA and AMD miners were released early in the month, with SRBMiner and Herominers pool support following. Node version 4.0.0 was released on January 18th, with 4.0.2 and 4.0.3 following. Exchanges, notably HotBit, required wallet rescans and temporary halted deposits/withdrawals during the transition.

### AgeUSD Stablecoin Protocol

Unveiled at the Ergo Summit on January 23rd and discussed in detail through the end of the month, AgeUSD is a two-token collateralized algorithmic stablecoin: StableCoins track a $1 USD peg, while ReserveCoins represent shares of the ERG reserve and allow speculators to absorb volatility and collect protocol fees. The system requires careful over-collateralization parameter selection to prevent undercollateralization during sharp price crashes. A key differentiator over Uniswap-style LP positions is that ReserveCoin holders benefit from upside autocorrelation without impermanent loss. The protocol was confirmed as not yet connected to mainnet, pending the hardfork. GitHub repository at github.com/Emurgo/age-usd was shared.

### Oracle Pools and Cardano Interoperability

Oracle Pools were a recurring topic across all four weeks, frequently discussed in the context of the Emurgo partnership and the Cointelegraph coverage. The pools aggregate data from multiple external sources directly on-chain without requiring an intermediary token layer (contrasting with Chainlink). Oracle Pool v1 was operational with Ergo Foundation members running data providers; Oracle Pool v2 was described as running for Dexy Gold with GORT rewards. The design was confirmed as compatible with both Ergo and Cardano smart contracts. Palmyra (then referred to as ZenGate) announced a Chainlink integration for proof of reserve in Winter Protocol for on-chain warehouse receipts, while noting plans to also build out Ergo oracles using token sale proceeds.

### Rosen Bridge and Cross-Chain Interoperability Roadmap

Cross-chain interoperability was discussed as a 2021 roadmap priority. The Gravity protocol gateways (Ethereum, Binance Chain, Waves, Tron) were confirmed as targets. Velvet fork sidechaining was mentioned. The Rosen Bridge concept — enabling ERG liquidity on other chains — was discussed, along with the broader point that Ergo's position as a "supplier of smart finance for all chains" should not provoke tribal reactions when projects use only parts of the Ergo ecosystem.

### Merge-Mined Sidechains and Network Speed

kushti outlined merge-mined sidechain constructions for Ergo, where mainchain miners earn rewards for delivering sidechain data. This differs from Bitcoin's BIP 301 with improved trustless pegging via smart contracts. Separately, the community discussed potential speed improvements — sub-blocks, microblocks, faster confirmation protocols, and layer 2 solutions. kushti noted that Ergo's concrete deployed application set (Rosen, Oracle Pools, Spectrum DEX contracts, lending, stablecoins) now makes on-chain optimization far more tractable than optimizing empty chains, and flagged a testnet available for tweaking parameters.

### ErgoMixer and Privacy

ErgoMixer (a non-custodial, non-interactive privacy tool using Sigma protocols) was frequently referenced for the practical use case of recovering funds from the closing Magnum wallet, which used a different derivation path from Yoroi. Approximately 90,000 ERG were in the mixer by late January. Users reported stuck funds when mixing amounts didn't align with ring requirements; the solution was to change ring size or wait for appropriate box sizes. No full node required for mixer operation.

### Storage Rent and Post-Emission Economics

Storage rent as Ergo's long-term miner incentive (analogous to Monero's tail emission but without new coin issuance) was explained multiple times across the month. It prevents permanent coin loss, discourages UTXO set spam (since sending unwanted tokens costs rent), and maintains miner incentives post-emission. Modeling of end-of-emission economics was described as approximately 2 years out.

### Wallet Ecosystem and Developer Tooling

Magnum wallet closure drove significant migration discussion. ErgoMixer provided a recovery path for legacy 15-word mnemonics not supported by Yoroi's derivation path. Ledger hardware wallet support was confirmed as in development (community grant to @jasondavies) but with no firm timeline. Yoroi mobile ERG support was also in progress. ErgoBootstrap was demonstrated as a Docker-based tool for local dApp development cluster deployments. ergo-ts was identified as having a double-spend bug (not tracking off-chain spent inputs) with a fix scheduled.

## Technical Q&A Worth Preserving

- **Q** (@ニック ᴏʟᴇᴜᴍ, 2021-01-04): How do I recover ERG from a pre-August 2020 wallet with a 15-word seed phrase not supported by Yoroi?
  **A** (@kushti): Use ErgoMixer v3.2.0+ to recover funds from the legacy Magnum mnemonic. No full node required. https://github.com/ergoMixer/ergoMixBack/releases — forum guide: https://www.ergoforum.org/t/magnum-wallet-closing-in-20-days/468/6

- **Q** (@UpwindStrategy, 2021-01-10): How will NIPoPoWs function after emission ends? Will miners still create PoW blocks?
  **A** (@kushti): Storage rent will replace the emission tail (analogous to Monero). Miners continue receiving transaction fees and storage rent, maintaining PoW difficulty. NIPoPoWs are mathematically dependent on PoW and cannot be replicated by PoS chains.

- **Q** (@UpwindStrategy, 2021-01-10): What determines max computational cost per block?
  **A** (@kushti): Analogous to Ethereum's gas limit — addresses the verifier's dilemma in block validation.

- **Q** (@CryptoTeller, 2021-01-23): Could AgeUSD supply ever exceed underlying reserves during a severe price drop?
  **A** (@kushti): Yes, which is why parameters must be chosen carefully with sufficient over-collateralization from the outset.

- **Q** (@S.Dbros, 2021-01-23): What's the difference between holding ReserveCoin versus just holding ERG?
  **A** (@kushti): ReserveCoin holders also earn fees collected by the AgeUSD bank, not just ERG price appreciation.

- **Q** (@c8, 2021-01-29): Can a PoW chain be pool-resistant if it has smart contracts?
  **A** (@c8, self-answered via analysis): No — smart contracts enable miners to form smart contract pools, defeating any pool-resistance mechanism at the protocol level.

- **Q** (@unknown, 2021-01-25): Getting `shutdownHook2` timeout errors running the Ergo node on Ubuntu 18.04.
  **A** (@kushti): Run with the `-Xmx3G` JVM flag: `java -jar -Xmx3G ergo.jar --mainnet -c mainnet.conf`

- **Q** (@Żeśo, 2021-01-23): Will sending unwanted tokens to users be possible/cheap as on Waves?
  **A** (@kushti): Storage rent makes sending unwanted tokens to others much more expensive on Ergo — a significant spam-prevention advantage.

- **Q** (@unknown, 2021-01-25): Why be a ReserveCoin provider if ERG price falls?
  **A** (community): Risk-adjusted return is positive if you believe ERG appreciates long-term. Unlike Uniswap LPs, ReserveCoin holders benefit from upside autocorrelation without impermanent loss.

- **Q** (@Albermonte, 2021-01-08): Why were two rapid tips from the same account rejected as double-spends?
  **A** (@kushti): The ergo-ts library is reusing the same one-time coin (UTXO) for different transactions without tracking off-chain spent inputs. The full node resolved this months ago; a library fix was scheduled. Workaround: use `/transactions/check` before posting via `/transactions` asynchronously, or implement input chaining (spending mempool outputs).

- **Q** (@unknown, 2021-01-29): ErgoMixer funds stuck after importing from Magnum — what to do?
  **A** (@MHS_SAM): Wait for the 250-ERG box to withdraw to Yoroi, then remix the remainder; alternatively, change the ring size (e.g. from 10 to 1 in sequence), but expect higher fees.

## Open Questions Entering Next Month

- **AgeUSD mainnet launch date**: Protocol ready but not yet connected to mainnet; dependent on post-hardfork stabilisation. Over-collateralization parameter values not yet published.
- **Exchange listings**: New listings "coming soon" per summit announcement but no specific exchanges, dates, or terms disclosed. Binance/Kraken/Coinbase presence remains absent.
- **Ledger hardware wallet ETA**: In development via community grant (@jasondavies) but no target date.
- **Yoroi mobile ERG support**: Described as arriving before Ledger support; no date given.
- **Oracle Pool v2 parameters and posting schedule configuration**: Whether posting schedules are configurable post-deployment remains unanswered.
- **Speed improvements / sub-blocks**: Testnet channel exists for parameter experimentation but no lead identified to drive the scientific study; block-height-dependent contracts create constraints that need mapping before parameters can safely change.
- **Dark pools on Ergo**: kushti identified the feasibility via Sigma protocols but indicated further careful study needed before commitment.
- **ergo-ts double-spend fix**: Scheduled "for next weeks" but not shipped within January.
- **REST API v1 design**: Community feedback accepted; implementation timeline unclear.
- **Bitcoin Schnorr activation**: Required for cross-chain atomic swaps with Bitcoin; timeline outside Ergo's control.
