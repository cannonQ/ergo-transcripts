# Ergo General Chat — Monthly Summary
**Chat:** Ergo General
**Month:** 2019-08 (covering W31–W35)

---

## Executive Summary

August 2019 was an early-ecosystem month dominated by user onboarding friction, node synchronisation challenges, and foundational community-building discussions. Token swap mechanics (EFYT → ERG), mining optimisation, and exchange listing activity were recurring themes, alongside the first stirrings of Ergo's decentralised application tooling with an alpha crowdfunding CLI and a detailed smart contract grants platform proposal.

---

## Key Topics This Month

### EFYT → ERG Token Swap
The 1:1 swap from EFYT (on Waves platform) to ERG mainnet was a persistent topic throughout the month. Users asked about execution times on the Waves DEX gateway, delays of up to 17+ hours were reported, and concerns arose about withdrawal errors. The team confirmed coins remain safe during gateway delays and that the swap would remain open for an extended period with no hard deadline. Tidex required approximately 100 confirmations before ERG appeared on the Ergo Explorer.

### Node Setup, Synchronisation, and Wallet Issues
A significant portion of community activity involved troubleshooting full node deployment. Common issues included wallets stuck at incorrect block heights, balance discrepancies from using the same mnemonic across multiple nodes, and slow initial sync even on capable hardware (i7, NVMe SSD, 100 Mbps). The recommended resolution was a full resync from genesis, waiting for CPU usage to drop before interacting with the wallet. No bootstrap snapshot was available at this time.

### Mining: Hashrates, Pools, and Centralisation Concerns
Community members shared GPU benchmarks throughout August: GTX 1060 ~12 MH/s, GTX 1070 ~20 MH/s (5× setup ~101 MH/s total), GTX 1080 Ti ~36 MH/s, 13× RX Vega ~280 MH/s. Suspicious mining activity was observed (blocks 23830–32838 with identical timestamps), prompting debate about 51% attack feasibility. Ergo's decentralised mining model — where pools require trust relationships between miners — was discussed as a structural constraint on traditional pool formation. Home miner viability versus large farm dominance remained an open concern. Network hashrate was reported at 1.17 TH/s in mid-August.

### Exchange Listings
Trading was live on the Waves platform and Tidex. The community discussed p2p2b integration (no timeline committed), Bisq support expected in September, Vite DEX, and CryptoBridge. Binance listing fees were estimated at approximately 500 BTC, considered potentially prohibitive. GJ.com (a Chinese exchange with a 400,000-user base) announced plans to list ERG in mid-September with a potential AMA and airdrop. CoinMarketCap listing status was queried but not confirmed.

### Community Crowdfunding and Grants Infrastructure
Substantive discussion emerged around decentralised community funding. A proposal modelled on ZF Grants but using ErgoScript contracts (P2S addresses) per proposal was detailed, featuring automatic refund mechanisms if funding goals are not met within time limits. A debate arose between a full smart contract approach (Jason Davies) and a simpler initial implementation (Dmitry). A Rust-based Ergo crowdfunding CLI reached alpha and was released on GitHub. T-shirt merchandise was discussed but met with scepticism regarding revenue potential in a ~900-member community.

### Hardware Wallet Integration
Trezor and Ledger integration complexity was investigated. The community generally preferred Trezor's security model but acknowledged Ledger's broader adoption. Wrapped BTC (WBTC-style) was raised as a potential cross-chain funding mechanism. No timeline for integration was established.

## Technical Q&A Worth Preserving

- **Q** (@Андрюхин, 2019-07-29): How do I view public/private keys after running ergo-3.0.4.jar?
  **A** (@Dmitry V): Open `http://localhost:9053` in your browser to access the wallet API interface.

- **Q** (@Андрюхин, 2019-07-30): How can I verify if my node is synchronised? The balance endpoint returns height 0.
  **A** (implied from context): Check the node info endpoint and compare `headersHeight` against current network height via the Ergo Explorer. Slow initial sync is expected; a restart may help.

- **Q** (@Unknown, 2019-08-05): How do I determine the transaction fee?
  **A**: Default fee is 1,000,000 nanoERG. When sending the maximum amount, deduct the fee from the value. Example: `wallet/payment/send` with `[{"address": "...", "value": 1000000000}]` sends 1 ERG with the default fee.

- **Q** (@Unknown, 2019-08-05): What does `registers` mean in `wallet/payment/generate`?
  **A**: See Ergo forum documentation on boxes and registers: https://www.ergoforum.org/t/ergo-terminology-a-box-and-a-register/32

- **Q** (@NC, 2019-09-02): Ergo node v3.0.7 not seeing recent incoming transactions despite valid on-chain confirmations; wallet `/balances` stuck at height 16855.
  **A** (@reb0rn21): Resync the blockchain and wallet state from genesis. Wait several hours after sync completes for CPU usage to drop before checking balance. Do not interact with the wallet during sync — only unlock at startup and let it complete.

- **Q** (@Madefi's 🦴, 2019-09-04): How long does a gateway transaction take from Waves DEX? Transfer not appearing after 35 minutes.
  **A** (@𝖒𝖊𝖓𝖙𝖆𝖑𝖎𝖘𝖙կշօ): Tidex requires approximately 100 confirmations before ERG appears on the Ergo Explorer. Actual time varies and gateway maintenance may cause additional delays. Coins remain safe on the gateway.

## Open Questions Entering Next Month

- **Bootstrap / fast-sync snapshots**: No mechanism existed to accelerate full node sync from genesis; all users must sync from block 0. Whether this would be addressed was unresolved.
- **51% attack feasibility**: Suspicious mining activity (identical block timestamps) raised concerns that were not conclusively resolved.
- **Home miner viability**: Whether Ergo's architecture could sustainably support small home miners versus pool/farm dominance remained open.
- **CoinMarketCap listing**: Queried but not confirmed during the month.
- **Lightweight wallet architecture**: How Ergo would implement a wallet separate from the full node (comparable to Grin's approach) was flagged but not answered.
- **Decentralised grants platform design**: Tension between a fully smart-contract-based approach and a simpler initial implementation was not resolved.
- **Trezor/Ledger integration timeline**: Complexity acknowledged but no roadmap committed.
- **Non-custodial participation without a full node**: How users could donate or participate in Ergo without running a node remained unclear.
