# Ergo General Chat — Monthly Summary: April 2019

---

## Executive Summary

April 2019 was defined by pre-mainnet preparation, with the community focused on two parallel tracks: finalising the Autolykos GPU miner for practical use, and clarifying the EFYT → ERG token swap and ErgoData sidechain mechanics. Technical onboarding dominated the chat, as early miners worked through CUDA compilation errors and node configuration challenges on testnet, while core contributors reinforced that mainnet launch remained the absolute top priority.

---

## Key Topics This Month

### Mainnet Launch Priority & Exchange Listings
The team consistently emphasised that mainnet launch superseded all other objectives. Exchange listing on at least one major liquid exchange was confirmed as an essential prerequisite before launch, though the EFYT token was unlikely to be listed outside the Waves DEX prior to mainnet. All secondary initiatives were explicitly deferred until after a successful launch.

### Autolykos GPU Miner: Compilation & Setup
A recurring theme throughout the month was the difficulty early adopters faced compiling and running the Autolykos GPU miner. Issues centred on CUDA/driver version incompatibilities, a libcurl dependency problem on Ubuntu, and user-defined literal operator errors. Andrey Andreev confirmed a working configuration using commit `7a89253b` on a GTX 1050 Ti 4GB after a libcurl downgrade. GPU mining (NVIDIA, 8GB VRAM) was confirmed significantly more efficient than CPU mining; CPU mining at testnet difficulty was deemed non-viable without GPU access.

### Node Setup & Testnet Mining
Multiple community members worked through full node setup, API key configuration, and testnet mining across the month. Key clarifications included: the 2–8 GB memory requirement in Autolykos refers to VRAM (not RAM); port forwarding and a static IP are not required to run a node; and remote GPU configuration is a viable workaround for server-only environments lacking local GPU hardware.

### EFYT → ERG Token Swap & ErgoData
The token swap timeline was estimated at approximately six months post-mainnet. 10% of first-year mining supply (~1,971,000 ERG) was designated as the swap source for EFYT holders. A related discussion clarified the ErgoData sidechain: originally conceived as a separate blockchain, it was repositioned as a sidechain to launch 2–3 years post-mainnet subject to demand. EFYT holders will receive ErgoData tokens (EFYTD) before mainnet, with the actual chain availability deferred; the token form (Ergo-native or Waves-based) remained to be decided.

### Autolykos Mining Pool Model
The Autolykos algorithm's requirement for miners to sign blocks with their own private keys was discussed as a fundamental departure from traditional mining pool models. This shifts payout responsibility onto the pools themselves, requiring new business model adaptations that were left open for future standardisation.

### Proof of Work Rationale
The choice of PoW over PoS was articulated as enabling lightweight, truly decentralised validation without functional compromise — a design principle consistent with Ergo's broader philosophy.

## Technical Q&A Worth Preserving

- **Q** (@李 思琦, 2019-04-25): GPU miner vs CPU miner — which is more efficient?
  **A** (@Dmitry): GPU is much more efficient.

- **Q** (@李 思琦, 2019-04-25): Does the "2–8 GB of memory" requirement in Autolykos refer to RAM or VRAM?
  **A** (@Dmitry): VRAM.

- **Q** (@李 思琦, 2019-04-25): Connected to testnet with `mining = true` but received no coins — configuration issue?
  **A** (@Dmitry): Requires GPU configuration for actual mining. Test coins can be sent manually if needed. Reference: https://github.com/ergoplatform/ergo/wiki/Mining

- **Q** (@李 思琦, 2019-04-25): No GPU available on server. Can mining proceed?
  **A** (@Dmitry): CPU mining at current difficulty is not viable. Option: configure a remote GPU on external hardware connected to the node.

- **Q** (@Rejinderi, 2019-04-25): Is port forwarding or a static IP required to run a node?
  **A** (@Unknown): Neither is required.

- **Q** (@Unknown, 2019-04-25): CUDA miner compilation errors on Ubuntu — any known fixes?
  **A** (@Andrey Andreev): A libcurl downgrade is required on Ubuntu. Commit `7a89253b` tested successfully on GTX 1050 Ti 4GB. See: https://github.com/ergoplatform/Autolykos-GPU-miner/issues/15#issuecomment-485994809

- **Q** (@Unknown, 2019-04-25): GPU works in other applications but Autolykos produces "Error checking GPU error" on Ubuntu (driver 396.24, CUDA 10.1).
  **A** (@Andrey Andreev): Driver/CUDA version incompatibility. Refer to the official NVIDIA compatibility matrix: https://docs.nvidia.com/deploy/cuda-compatibility/index.html#binary-compatibility__table-toolkit-driver

- **Q** (@Unknown, 2019-04-08): Why would mining pools require private keys?
  **A** (@Unknown): Autolykos requires miners to sign blocks with their own private keys. Mining pools must therefore handle fair payouts themselves, fundamentally changing traditional mining business models.

- **Q** (@Unknown, 2019-04-20): Will EFYT holders receive ErgoData tokens (EFYTD)? How will address mapping work across the 1-year swap period and 2–3-year sidechain development window?
  **A** (@Unknown): Yes — EFYTD will be sent to EFYT holders before mainnet. The token form (Ergo-native or Waves-based) is to be decided before launch. The ErgoData chain itself will be available 2–3 years post-mainnet; a sidechain architecture is preferred over maintaining dual independent blockchains.

## Open Questions Entering Next Month

- **GPU Miner Idle State**: A reported behaviour where the Autolykos GPU miner finds a solution then idles at 0% GPU utilisation was unresolved at month end, pending log review by the development team.
- **EFYT Address Mapping**: The precise mechanism for tracking EFYT holder addresses across the 6-month swap window and the 2–3-year ErgoData sidechain wait period was not fully specified.
- **Mining Pool Business Models**: Exact contractual and payout mechanisms for pools operating under Autolykos's private-key signing requirement remain open for future standardisation.
- **EFYTD Token Form**: Whether ErgoData tokens distributed to EFYT holders pre-mainnet will be Ergo-native or Waves-based had not been decided by month end.
- **GPU Miner Compilation Root Cause**: While workarounds were identified, the underlying cause of Autolykos GPU miner CUDA errors across varying Ubuntu/driver configurations was not fully resolved.
