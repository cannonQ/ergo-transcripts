# Ergo General Chat — Monthly Summary
**Month:** August 2024

---

## Executive Summary

August 2024 was dominated by three converging themes: core protocol advancement (Ergo 6.0.0-alpha release, Sigma Rust 0.28, node v5.0.22/5.0.23), expanding cross-chain infrastructure (Rosen Bridge EVM testnet, rsnBTC live on mainnet, Ethereum↔Ergo testnet validation), and ecosystem maturation through new DeFi primitives (Gluon Gold soft launch, ChainCash mainnet demo, duckpools IDO). Community discussion reflected growing tension between technical readiness and adoption/marketing investment, with sustained debate on UX improvement across dApps, wallet onboarding, and storage rent presentation.

---

## Key Topics This Month

### Ergo 6.0 Protocol Development
Ergo 6.0 moved from concept to active devnet testing across August. kushti ran the first local 6.0 devnet (W34), merged multiple PRs into the 6.0 candidate branch (W32), and released v6.0.0-alpha0 for public devnet testing on 2024-08-26 (W35). Key improvements target: simplified cross-chain integration (including trustless Bitcoin relay demonstrated at ErgoHack VIII), rollup support, improved blockchain structure access for Lithos Protocol, and expanded cryptographic protocol support. The Lithos team completed initial client development but remains blocked pending 6.0 devnet availability. Community members were invited to participate in testnet parameter tweaking — including block time experiments — though a coordinator willing to lead the effort was needed. Sub-blocks/microblocks were identified as "the hot topic" on the scalability pipeline but remain work-in-progress, with an active GitHub branch (`weak-blocks`) and forum discussion ongoing.

### Rosen Bridge Expansion
Rosen Bridge made major strides this month. rsnBTC (bridged Bitcoin) went live on Ergo mainnet (announced 2024-08-09). The first end-to-end Ethereum↔Ergo bridge testnet transaction was completed successfully (2024-08-12), with full transaction documentation. EVM support was deployed on private testnet by W32, with test transactions imminent by end of month. Armeanio reported parallel work on BigInt refactor, UI improvements, and eRSN token enhancements. A UI bounty program was launched on GitHub to encourage community contributions. The longer-term vision of Rosen as a universal cross-chain interoperability hub — supporting Bitcoin, Cardano, EVM chains, and eventually Solana — was articulated across multiple discussions.

### Gluon Gold Launch & DeFi Protocol Mechanics
Gluon Gold soft-launched in W32, introducing fission/fusion/transmutation mechanics for gold-pegged stablecoin issuance. GAU (stablecoin) and GAUC (reserve token) became tradeable on CruxFinance DEX. Bruno Woltzenlogel Paleo provided detailed explanations of the fee structure (decreasing protocol fees 0.5%→0%, perpetual 0.1% oracle fees, optional 0–0.4% UI fees) and reserve ratio mechanics. A significant UX debate emerged: the fusion interface accepts ERG input rather than GAU/GAUC amounts, which confused users — Bruno acknowledged this as intentional but flagged for improvement, with a GitHub issue opened and SDK changes potentially required. Bruno also announced the Gluon Gold SDK as a prerequisite for the broader roadmap, with contributors earning gluons and membership in The Stable Order.

### ChainCash Mainnet Demo & Use Cases
kushti demonstrated the first mainnet ChainCash reserve, note issuance, and spending transaction via API (2024-08-11), a significant milestone for the trust-based currency framework. Across W32–W35, the community explored ChainCash's practical applications: resource-constrained communities issuing LETS-style local currencies, commodity producers (e.g., Ghana mining cooperatives) issuing notes backed by production instead of seeking loans, and multi-currency businesses requiring clearing mechanisms rather than a unified currency. Glasgow provided a detailed example chain: commodity note → government tax payment → Cedi conversion → oil purchase → equipment trade. ChainCash's CCIP-1 extension (NFT-gating for notes) was proposed by kushti in W34. The community identified a need for dApp-layer tooling to make ChainCash accessible beyond API-level usage.

### SigmaUSD / duckpools Liquidation Event
A cascading liquidation event occurred on 2024-08-04–05 following a ~15% ERG price drop. No malicious attack was detected; the cause was natural market downturn combined with off-chain bot congestion. Lenders profited ~1.5% overall despite some individual liquidations at a loss. duckpools announced a v2 roadmap prioritizing borrower safety dashboards, risk alerts, arbitrage bots, and educational guides. The IDO for duckpools phase 2.2 reopened (W32), with core devs agreeing to complete goals retroactively if funding fell short. This event also sparked broader discussion about stablecoin design philosophy — community consensus favoring permissionless synthetics (SigmaUSD, Gluon Gold, Dexy Gold) over custodial stablecoins (USDC) that require freeze functions incompatible with the eUTXO model.

### Autolykos Mining Table Size Increase
The Autolykos mining memory table increased to 4.3 GB during W33, causing widespread mining disruption for users running NBMiner. kushti recommended switching to GMiner or T-Rex. Users with GPUs below 6 GB VRAM (3060 Ti, 3070, 1660 Super) were effectively excluded from mining. MinerStat's displayed DAG size (3.55 GB) was identified as incorrect; the community-maintained DAG size spreadsheet was the authoritative reference.

### Ecosystem Project Coordination & UX
An ongoing theme throughout August was the friction between developer-centric tooling and end-user experience. Discussions covered: whether a Jira-like centralized project board should exist (W31, no consensus); how to present storage rent to non-technical wallet users (W34 — "Storage Rent Due" preferred over "Mineable," with approximate dates and consolidation UI); and whether Ergo dApps suffer from 1990s-era aesthetics and overly technical terminology that discourages adoption. sigmaverse.io and the Grist project management board were identified as the closest existing coordination tools. qx() proposed extracting dev update content into a centralized database to avoid double-reporting.

### ErgoHack IX & Community Events
ErgoHack IX was announced for October 2024 (W33), with registration opening the following week. Focus areas include developer tools, dApps, and marketing initiatives. The Hash It Out AMA series was revived (W31), featuring CW, qx(), Quokka, and Dan Friedman, covering Palmyra/Chainlink integration, Rosen Bridge status, Trade House order books, DDoS resilience, and fee market mechanics. The ErgExplorer Annual VPS Fund reached its 530 ERG goal with 30 contributors (W33). kushti proposed a Twitter Spaces schedule for September: building applications on Ergo (Sep 5) and Ergo's memetic identity (Sep 12).

## Technical Q&A Worth Preserving

- **Q** (@Alex, 2024-07-29): Is there an existing board showing all Ergo projects with backlogs, advancement tracking, and contribution opportunities?
  **A** (@qx()): sigmaverse.io provides the best ecosystem overview with direct links to project roadmaps and Git commits. A centralized board would require devs to double-report; a better approach would extract data from existing dev updates automatically.

- **Q** (@Alex, 2024-07-29): Why use the UTXO model instead of a simpler account-based system?
  **A** (@Luivatra): The UTXO model allows transactions to only care about the specific boxes involved, making the billions of other boxes irrelevant. Account-based systems require tracking global account state, which scales poorly. Boxes are uniquely identified, spent exactly once, and contain goods with linked keys — a wallet is a warehouse storing keys to multiple boxes across addresses.

- **Q** (@Unknown, 2024-08-06): If I fission 50 ERG into GAU/GAUC, then gold price rises, then I fuse back — do I get more ERG?
  **A** (@Bruno Woltzenlogel Paleo): No — you receive the same amount of ERG minus fees, assuming no other transmutations occurred. Gold price only affects transmutations, not fission/fusion. Holding only one token (GAU or GAUC) exposes you to gold price movement.

- **Q** (@Unknown, 2024-08-06): Why can't I mint ERG directly into GAU or GAUC alone?
  **A** (@TMR): To maintain a healthy reserve ratio. Simultaneous minting of both tokens ensures balanced exposure and prevents one-sided reserve drain.

- **Q** (@Man ish, 2024-08-11): What advantages does Sigma 6.0 offer non-developers?
  **A** (@kushti): Simpler cross-chain integration (e.g., trustless Bitcoin relay), rollup support, improved blockchain structure access for Lithos Protocol, and various protocol fixes enabling the next generation of dApps.

- **Q** (@Unknown, 2024-08-11): What are ChainCash's main use cases?
  **A** (@kushti): (1) Remote communities with labor/resources but no money — trade via collateralized notes or trust-based LETS internally. (2) Commodity producers issuing notes backed by production instead of seeking complex loans. (3) Multi-currency businesses (e.g., BRICS context) needing clearing mechanisms rather than a unified currency.

- **Q** (@Mike, 2024-08-16): All mining rigs stopped — is something wrong with the Ergo network?
  **A** (@kushti): The Autolykos mining memory table size increased to 4.3 GB. Switch from NBMiner to GMiner or T-Rex. GPUs below 6 GB VRAM are no longer sufficient.

- **Q** (@bitcoincashautist, 2024-08-15): Why hasn't USDC been bridged to Cardano or Ergo?
  **A** (@Glasgow): Neither Cardano nor Ergo support the freeze function that USDC requires. A new standard would be needed. Permissionless synthetic stablecoins (Gluon Gold, Dexy Gold, SigmaUSD) are better aligned with the eUTXO model philosophy.

- **Q** (@DiscoverErgo, 2024-08-20): Doom runs on Cardano's Hydra — do we have similar scaling solutions?
  **A** (@kushti): Hydra can be built on Ergo but is only suitable for applications with a defined set of trusted state signers.
  **A** (@Armeanio): Oracle Pools make the most sense as a scaling approach currently.

- **Q** (@Flying Pig, 2024-08-22): How should wallets present the storage rent feature to non-technical users?
  **A** (@Flying Pig): Use "Storage Rent Due" instead of "Mineable," show approximate dates, provide a consolidated summary of all boxes with rent due dates, and add an optional consolidation button. A dedicated explainer page is needed.

- **Q** (@Shiv Chauhan, 2024-08-26): Can Ergo still be mined with a 4 GB GPU?
  **A** (@Unknown): No — the memory table now exceeds 4 GB. A minimum of 6 GB VRAM is required. MinerStat's displayed DAG size (3.55 GB) is incorrect; refer to the community DAG size spreadsheet.

- **Q** (@Alex, 2024-08-31): Is there a dApp to create ChainCash notes? How do trust-backed notes work?
  **A** (@kushti): ChainCash is an "assembly language layer for monetary systems" — dApp-layer tooling is needed for accessible applications. Low-hanging-fruit candidates include local community LETS systems deployable as layer 2 solutions.

## Open Questions Entering Next Month

- **Sub-blocks implementation timeline**: Identified as the primary scalability priority, with an active `weak-blocks` GitHub branch, but no committed deployment date, design finalization, or testnet launch window. A coordinator to lead community testnet participation is still needed.

- **Block height vs. timestamp contract dependencies**: Reducing block time requires auditing all deployed contracts for block height dependencies. Scope of impact has not been quantified and could constrain how aggressively block speed can be increased.

- **Rosen Bridge EVM mainnet launch**: Private testnet deployed by end of August; no public launch date announced. BigInt refactor and UI work running in parallel.

- **Gluon Gold fusion UX**: The fusion interface requiring ERG input (rather than GAU/GAUC amounts) confused users; a GitHub issue was opened but SDK changes may be required. Resolution timeline unclear.

- **ChainCash dApp layer**: kushti demonstrated API-level functionality but identified a gap between the protocol and accessible end-user tooling. No specific dApp candidates or development timeline announced.

- **Storage rent UX standardization**: Flying Pig proposed auto-consolidation on every wallet interaction; Aberg disagreed. No resolution reached among wallet developers.

- **Messaging platform**: Community discussed moving away from Telegram (IRC, Matrix 2.0, DarkIRC considered) but no consensus reached on a viable alternative that preserves ecosystem cohesion.

- **SigmaUSD structural improvements post-liquidation**: duckpools proposed risk dashboards and v2 features but full implementation is funding-dependent. No protocol-level changes to prevent cascade liquidations announced.

- **Ledger app dev mode**: Ergo's Ledger app has remained in developer mode for 1.5+ years with no update on path to official release.

- **qx() AMA "huge secret"**: A significant revelation was reportedly cut from the 2024-08-22 live AMA. Details have not been disclosed.
