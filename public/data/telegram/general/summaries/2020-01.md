# Ergo General Chat — Monthly Summary: 2020-01

---

## Executive Summary

January 2020 was an early community-building month for Ergo, dominated by two parallel tracks: strategic discussions around exchange listings and market access, and hands-on technical experimentation with node deployment on resource-constrained hardware. The month also saw foundational governance conversations emerge, with community members exploring how ErgoScript could enforce ICO escrow and roadmap accountability — an early signal of Ergo's programmability ambitions.

---

## Key Topics This Month

### Exchange Listings & Market Access
The community invested significant energy debating which major exchanges to target — Bittrex, Poloniex, OKEX, Bitfinex, Kraken, Huobi, and Binance all featured in discussion. A recurring theme was the unreliability of CoinMarketCap volume figures compared to CoinGecko, and uncertainty around actual listing fees and regulatory documentation requirements. The Ergo Foundation confirmed it had already contacted multiple exchanges and was resubmitting applications. Current liquidity remained thin, with Tidex and Waves.exchange identified as the most viable options for acquiring ERG. The EFYT token swap (Waves-based EFYT to native ERG at 1:1) was ongoing but seeing weak demand.

### Node Deployment on Raspberry Pi
An extended multi-week thread explored running Ergo full and light nodes on Raspberry Pi hardware (1GB RAM Model B+). The key finding was that full nodes are too resource-intensive for 1GB RAM devices, but with JVM memory tuning (`-Xmx512m`) a sync could be achieved. filippounits successfully deployed a node and published a configuration gist. Comparative notes from LTO/Waves Raspberry Pi deployments were also shared. This thread surfaced the broader need for a proper light node implementation to support decentralised participation on low-resource hardware.

### Mining Setup & Pool Architecture
Several community members worked through GPU mining configuration questions, including CUDA compute architecture parameters for RTX 2080 Ti cards, Java version requirements (Java 8 only for official JRE support), and whether a hot wallet is mandatory for pool operation (confirmed: yes). A more advanced proposal emerged for a distributed mining pool design using collateralized private keys issued by the pool to individual miners, though implementation details remained exploratory.

### ErgoScript for ICO Governance
A notable governance discussion arose around using ErgoScript to create escrow contracts for ICO releases tied to roadmap milestones — essentially enforcing community consensus (e.g., 51% agreement) before funds are unlocked, as a structural deterrent to exit scams. This was highlighted as a capability not widely available in other smart contract platforms. Questions about whether Ergo needed formal on-chain governance (à la Tezos) were raised but not resolved.

### Community Platform & Infrastructure
The community expressed a clear preference for Discord over Slack. The official Ergo Foundation website launched at ergoplatform.org. The 10M ERG supply milestone was reached. Ergo's academic-to-production pipeline was noted approvingly: published research feeds directly into live implementation rather than diverging from it.

## Technical Q&A Worth Preserving

- **Q** (@Unknown, 2020-01-W03): "I have ergo-tokens issued in Waves platform before Ergo starts, can I convert them to native Ergo coins? Is it still possible?"
  **A** (@Kookster): "There is an EFYT/ERG market on Waves where you can trade your EFYT for ERG at 1:1."

- **Q** (@Unknown, 2020-01-W03): "Hi guys, I'm trying to set up a full node on Linux. How long does it take to sync and load all the blocks on a 120Mbps internet?"
  **A** (@Unknown): "Depends on hard drive, should be around 1 hour."

- **Q** (@Unknown, 2020-01-W03): "CUDA_COMPUTE_ARCH has 60 as default. I have a RTX 2080 Ti, should I change that parameter?"
  **A** (@Unknown): "Should be autodetected with the latest version of the miner."

- **Q** (@Unknown, 2020-01-W03): "Do I need Java 9 on Windows?"
  **A** (@Unknown): "Java 8 is the only version with official JRE support; versions 9–11 lack security updates and are experimental."

- **Q** (@god, 2020-01-W03): "When there is no private key in my wallet, can I still get the task normally? Must Ergo mining use a hot wallet?"
  **A** (@scalahub): "Yes, as far as I know, you must have a hot wallet."

- **Q** (@Unknown, 2020-01-W04): "Is it possible to create escrow mechanisms in ICOs using ErgoScript to release payments based on roadmap completion?"
  **A** (@Unknown): "ErgoScript's flexible capabilities enable governance-enforced ICO contracts with community voting thresholds (e.g., 51% agreement on roadmap completion) — a novel application not widely seen in other smart contract platforms."

- **Q** (@Unknown, 2020-01-W04): "What are the specific things to test on Raspberry Pi Model B+ (1GB RAM) for running an Ergo node?"
  **A** (@filippounits): "Running on 1GB RAM will cause issues for a full node. With `-Xmx512m` flag, sync is achievable; light nodes are the proper solution for RPi-class hardware."

## Open Questions Entering Next Month

- **Exchange listing priority**: No consensus on which single tier-1 exchange to pursue first; tradeoffs between Bittrex, Poloniex, OKEX, Kraken remain unresolved.
- **Light node roadmap**: Official light node implementation timeline not confirmed; community experimentation on Raspberry Pi highlighted the gap.
- **EFYT swap demand**: Swap from Waves-based EFYT to native ERG proceeding but with weak uptake — unclear if this represents a communication gap or genuine disinterest.
- **On-chain governance model**: Whether Ergo will adopt formal on-chain governance voting (à la Tezos) remains undecided; ErgoScript-based escrow governance proposed but not scoped.
- **GUI/UX investment**: Whether the core team will allocate resources to wallet UX improvements flagged as a gap, but no decision documented.
- **Mining pool hot wallet requirement**: Whether miners can construct coinbase transactions independently (without a pool-managed hot wallet) remains technically unresolved.
