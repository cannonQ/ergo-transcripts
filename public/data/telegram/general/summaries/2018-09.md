# Ergo General — Monthly Summary
**Month:** September 2018

---

## Executive Summary

September 2018 was a foundational pre-mainnet month for Ergo, dominated by early testnet experimentation, node setup, and community onboarding. The release of testnet v1.6.1 gave early adopters their first hands-on experience with the network, while technical discussions clarified the emission policy, initial token distribution, and the readiness of ErgoScript. The primary outstanding item was the finalisation of the Autolykos proof of work algorithm, blocking the path to testnet 2 and the mainnet prototype.

---

## Key Topics This Month

### Testnet v1.6.1 & Node Operations
The release of testnet v1.6.1 marked the month's most concrete milestone, giving community members their first opportunity to run nodes and experiment with the network. Early adopters worked through configuration requirements — particularly the need to use the correct `application.conf` template from GitHub and the default API port of 9052 — and reported successful node startups and chain synchronisation. Docker instructions were flagged as pointing to an outdated P2P port and noted for update. An additional testnet version was planned before mining would become available to the public.

### ErgoScript (Sigma State Language)
A recurring point of reassurance throughout the month was the confirmation that the Sigma state language (ErgoScript) was already live on testnet. Working examples for asset atomic exchange contracts were available in the `sigmastate-interpreter` repository, demonstrating that smart contract functionality was not blocked on the PoW finalisation work. This separated ErgoScript readiness from the broader timeline uncertainty around mining.

### Emission Policy & Token Distribution
Community questions around the initial EFYT token distribution to Waves holders prompted a clarifying discussion on what constitutes a "pre-mine." The consensus was that the Waves-based airdrop mechanism, alongside founders' rewards and team allocation, operates outside the standard PoW emission process by design. The CoinMarketCap listing was acknowledged as carrying outdated emission data and requiring a full refresh. The emission curve documentation was shared as the authoritative reference.

### Proof of Work & Roadmap
Autolykos PoW finalisation was identified as the last major component before testnet 2 and the mainnet prototype phase could proceed. Version 1.7 was scoped to include a new fee approach prior to that transition. No firm timeline for mining availability was confirmed during the month.

## Technical Q&A Worth Preserving

- **Q** (@Unknown, 2018-09-26): Is the initial EFYT token distribution to Waves holders technically a "pre-mine" given it bypasses PoW minting?
  **A** (@Unknown): Founders' rewards and team allocations are also not mined via PoW; whether this constitutes a pre-mine depends on how one defines initial distribution. These mechanisms are distinct from standard PoW emission by design.

- **Q** (@Unknown, 2018-09-26): Node fails to connect to testnet with a "Failed to connect to:" error — what is wrong?
  **A** (@𝖒𝖊𝖓𝖙𝖆𝖑𝖎𝖘𝖙կշօ): The configuration file must be set correctly. The latest config template is available on GitHub; the default API port is 9052.

- **Q** (@Unknown, 2018-09-26): Is there documentation available for running a testnet node?
  **A** (@𝖒𝖊𝖓𝖙𝖆𝖑𝖎𝖘𝖙կշօ): All available documentation is on GitHub; the `-help` flag provides implementation-level details.

- **Q** (@Unknown, 2018-09-26): Will smart contracts be available during testnet 2 / the mainnet prototype?
  **A** (@Unknown): ErgoScript (the Sigma state language) is already available on the current testnet, with working examples for asset atomic exchange contracts already accessible in the repository — no need to wait for testnet 2.
