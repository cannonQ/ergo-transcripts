# Ergo Developer Chat — Monthly Summary
**Chat:** Ergo Developer | **Month:** 2023-07

---

## Executive Summary

July 2023 was defined by two parallel tracks: core infrastructure maturation (UTXO set snapshots, the Ergonnection P2P library, and node tooling improvements) and a wave of ecosystem tooling releases (Satergo v1.6.0, Erg Explorer updates, Ergasia job board, and GreasyCEX Mission #1 preparation). Foundational developer questions around ErgoScript toolchain setup, transaction mempool behaviour, and mining algorithm design were recurring themes throughout the month, reflecting a growing developer base onboarding to the platform.

---

## Key Topics This Month

### UTXO Set Snapshots & Node Bootstrapping
Nodes running v5.0.12 generated the first UTXO set snapshot at block height 1,044,479, unlocking snapshot-based bootstrapping on mainnet for the first time. This was identified as a significant milestone for reducing node sync time. NIPoPoWs were also discussed in this context — kushti clarified that they only make practical sense when paired with full block pruning (e.g., `utxoBootstrap` mode), and noted that the testnet genesis block ID could be corrected in a subsequent release.

### Ergonnection P2P Library
Aberg's Java-based peer-to-peer node communication library, Ergonnection, was the most sustained technical development thread of the month. The library progressed from initial release through community testing, with issues resolved around Blake2b256 transitive dependency management, connection timeouts after ~10 minutes of inactivity (addressed via modifier request messages and reconnection logic), and occasional "incorrect magic bytes" errors likely attributable to outdated peer software. Block header object support was added mid-month. The library was approaching Maven Central publication by month's end, pending further external testing.

### ErgoScript Toolchain & Language Support
A recurring onboarding question throughout the month was which languages could be used to write Ergo smart contracts. The answer was consistently clarified: **on-chain validators must be written in ErgoScript only**; off-chain transaction construction uses AppKit (Scala/Java), Sigma-Rust, or language wrappers (Go, JavaScript). The sigma.js compiler package was integrated into Fleet SDK, enabling browser-based ErgoScript compilation — noted as a meaningful tooling advancement. A developer with a Cardano/eUTXO background was pointed toward the `ergoscript-by-example` GitHub repository and official docs. Sigma-state Maven dependency version selection (Scala 2.11/2.12/2.13 suffixes) remained an open question.

### Storage Rent & Creation Height Enforcement
EIP-0039 (Monotonic creation height) was discussed in the context of preventing rent manipulation via arbitrary creation height setting. It was identified that some historical Spectrum bots had set creation heights to zero, which could have impacted users. The ergoforum thread on storage rent mechanics was referenced for deeper context.

### Mempool Transaction Behaviour
Aco raised concerns about transactions disappearing from the mempool, particularly during rapid redemption sequences on ErgoPad. The root cause was diagnosed as an application-level timing issue: when transactions are submitted too rapidly, subsequent transactions reference inputs that the node has not yet registered as spent from the prior transaction, causing them to be dropped. This was not a protocol-level flaw but an ErgoPad code issue requiring an update.

### Autolykos Mining Algorithm Optimisation
An extended debate arose over whether Autolykos memory bandwidth requirements could be increased to reduce GPU core power usage while preserving ASIC resistance. The technical consensus was that algorithm efficiency improvements alone do not improve per-hash profitability because difficulty adjusts accordingly — the only meaningful lever is increasing memory bandwidth dependency to shift the compute/memory balance. Any such change would require a hard fork, miner and developer alignment, and a coordinated research process. No current ASIC threat was identified as justifying an immediate change; the discussion was deferred without a decision.

### Ecosystem Releases & Projects
Several projects reached significant milestones in July. **Satergo v1.6.0** shipped with a Windows installer, ErgoPay dApp/DEX support (including Spectrum DEX swaps), CoinGecko fiat conversions, improved transaction history, and Indonesian/Malaysian translations. **Erg Explorer** added financial data for tokens, 24h volume and gain/loss tracking, transaction confirmation notifications, and Nautilus wallet integration. **Ergasia**, a decentralised job board for the Ergo ecosystem, entered beta with a temporary WordPress frontend while its Next.js frontend and smart contract backend were in active development. **GreasyCEX Mission #1** completed tokenomics finalisation, NFT minting, and Spectrum DEX listing, with a Twitter Spaces AMA planned ahead of launch.

## Technical Q&A Worth Preserving

- **Q** (@kushti, 2023-07-10): Does converting to `BigInt` solve integer overflow in price calculations using `Long` values?
  **A** (@kushti): Yes. The expression `val price: Long = (reserveIn.toBigInt * precisionFactor) / hodlCoinsCircIn` resolves the overflow without error by performing intermediate arithmetic in `BigInt` before casting back to `Long`.

- **Q** (@Aberg, 2023-07-12): Why is ZeroMQ needed if nodes already broadcast transactions over the TCP P2P layer?
  **A** (@Aberg): The TCP layer already exists and is used by all nodes. You can listen for `Inv` messages and request modifiers directly. The main concern is node weight — adding ZeroMQ introduces additional system dependency and complexity.

- **Q** (@Cheung Wai Lok, 2023-07-15): Can I write smart contract validators in Rust or Go, or only ErgoScript?
  **A** (@Pulsarz): **Only ErgoScript for on-chain validators.** Rust, Go, and JavaScript are used for off-chain transaction construction via AppKit, Sigma-Rust, or their respective language wrappers.

- **Q** (@Cheung Wai Lok, 2023-07-15): What are good resources for ErgoScript development coming from a Cardano/eUTXO background?
  **A** (@Flying Pig, @Cheung Wai Lok): The official ErgoScript docs at `docs.ergoplatform.com` and the `ergoscript-by-example` GitHub repository, which provides clear examples of core ErgoScript functionality.

- **Q** (@Aberg, 2023-07-17): Why are non-ERG tokens in `ErgoBoxCandidate` called `additionalTokens` rather than just `tokens`?
  **A** (@kushti): "Additional" or "secondary" reflects that ERG is the primary asset. The naming distinguishes native ERG from non-native tokens.

- **Q** (@Aberg, 2023-07-17): Should `additionalRegisters` be renamed to just "registers"?
  **A** (@kushti): No. All box fields — value, script, tokens, and R0–R9 — are technically registers. R0–R3 have fixed semantics; R4–R9 are user-defined. The prefix "additional" distinguishes the custom registers from the protocol-defined ones.

- **Q** (@Luivatra, 2023-07-26): `java.lang.NoClassDefFoundError: org/bouncycastle/jcajce/provider/digest/Blake2b$Blake2b256` when using Ergonnection.
  **A** (@Aberg): The JAR must include transitive dependencies (bouncycastle, sigmastate). Solutions: (1) copy dependency declarations from Ergonnection into your own project; (2) publish to Maven Local using the `maven-publish` plugin; (3) use the `shadow` plugin to build a fat JAR with all dependencies bundled.

- **Q** (@Luivatra, 2023-07-26): Socket connection closes after ~10 minutes of inactivity.
  **A** (@Aberg): Requesting modifiers prevents timeout-based disconnection. However, peer eviction can still occur unpredictably — implement reconnection logic. Optionally check whether your node's peer settings permit persistent connections.

- **Q** (@Aco Šmrkas, 2023-08-02): Why do transactions disappear from the mempool? This doesn't seem to happen on other blockchains.
  **A** (@Luivatra): This is an application-level issue, not a protocol flaw. When rapid sequential transactions are submitted before the node has registered the outputs of the prior transaction, the subsequent transactions reference already-spent inputs and are rejected. The ErgoPad code is outdated and needs updating to account for this ordering requirement.

- **Q** (@Pulsarz, 2023-08-02): Can Autolykos efficiency improvements reduce GPU power consumption while maintaining ASIC resistance?
  **A** (@Pulsarz): Algorithm efficiency gains alone don't improve per-hash profitability because difficulty adjusts. The only meaningful lever is increasing memory bandwidth dependency to shift the core/memory compute balance. This requires a hard fork, miner and developer consensus, and coordinated research — no current ASIC threat justifies immediate action.

## Open Questions Entering Next Month

- **Sigma-state Maven dependency versioning**: Which Scala suffix (2.11/2.12/2.13) should be used for `sigma-state-interpreter`? Status of refactoring to decouple ErgoTree from the ErgoScript compiler remains unclear.
- **Ergonnection "incorrect magic bytes" errors**: Root cause of `[31, -4, 74, -43]` vs `[1, 0, 2, 4]` mismatches during peer connections unconfirmed; likely outdated peer software but requires message-level inspection to confirm.
- **Node restore feature (PR #1417)**: kushti proposed replacing the disabled restore feature with a warning + confirmation flow. Community feedback and final decision not yet captured.
- **Autolykos memory bandwidth increase**: Technical feasibility established but no roadmap commitment made; requires miner and developer consensus and hard fork planning.
- **EIP-0021 token registry decentralisation**: Current registry model acknowledged as centralised; nemo reportedly working on an alternative approach with no timeline or specification shared.
- **Block height vs. timestamp effects of faster block times**: How speed improvement proposals would affect contracts with assumptions about block height ↔ timestamp correlation requires careful security analysis; not yet formally addressed.
- **ZeroMQ vs. TCP P2P for mempool event subscriptions**: Whether ZeroMQ is genuinely necessary or whether the existing TCP P2P layer can be adapted remains an open design question.
- **`ergo-lib-go` completeness**: Which functions remain unimplemented in the Go Sigma-Rust wrapper; whether transaction serialisation and validation are achievable with the current state.
- **API filtering by token ID and address pair**: No confirmed API endpoint exists to filter transactions by token ID and sender/recipient combination.
