# Ergo Developer Chat — Monthly Summary
**Chat:** Ergo Developer
**Month:** 2022-12

---

## Executive Summary

December 2022 saw focused developer activity around the Node v5.0 deployment and its downstream effects on mining pool payments, alongside a significant Satergo wallet release and early discussions on the EIP process. Technical threads on contract design (P2S vs P2PK encoding, Dexy Protocol refactoring, and AppKit type handling) dominated the month, with infrastructure concerns around GitHub dependency risk and Akka library licensing surfacing as longer-term open questions.

---

## Key Topics This Month

### Node v5.0 & Mining Pool Payment Bug
A bug was identified early in the month where EIP-27 outputs in payment transactions were being initialised with creation height = 0, causing failures in mining pool payment processing under v5.0. [@kushti, msg#9179, 2022-12-06] investigated and flagged the root cause, though full resolution was not captured within the month. A related issue — data inputs being inadvertently spent during block assembly, causing script verification failures — was debugged by [@kushti] in the final week of the period.

### P2S vs P2PK for Message/Attachment Encoding
A substantive design discussion emerged around whether context variables and message/attachment fields should be standardised across P2S and P2PK output types. The consensus reached was that P2S contracts are too flexible for standardised encoding assumptions — order book contracts and other constructions may use registers in non-standard ways — making P2PK and multisig outputs the more practical target for any message attachment standard. The possibility of adding new output fields via soft-fork (dependent on serialiser flexibility) was also noted. [@Cheese, msg#9517–9525, 2022-12-19; @kushti, msg#9564, 2022-12-19]

### Dexy Protocol Contract Refactoring
[@scalahub] continued active work on the Dexy Protocol, splitting the LP contract into smaller components following review comments from [@kushti]. A separate issue was raised regarding the absence of true inclusion height in boxes, filed as GitHub issue #1214. [@scalahub, msg#9612, 2022-12-21; msg#9789–9790, 2022-12-23]

### Satergo v1.5.0 / v1.5.1 Release
[@Aberg] shipped a major Satergo release featuring a complete UI redesign (by Nadi), automatic restoration of derived addresses, keyboard-only navigation in setup, node auto-update, and custom stylesheet support. A follow-up v1.5.1 patch was also released. This was the most significant ecosystem tooling release of the month. [@Aberg, msg#9802, 2022-12-23; msg#9868, 2022-12-28]

### AppKit 5.0.0 Type Mismatch Regression
[@scalahub] reported a type mismatch regression in AppKit 5.0.0 where `ErgoValue[Coll[Byte]]` construction failed due to a `Coll[scala.Byte]` vs `Coll[java.lang.Byte]` conflict. [@Luivatra] identified the new `ErgoValueBuilder` as the correct workaround. The full scope of affected type bindings remained unclear at month end.

### EIP Process & Community Governance
Discussion emerged around lowering barriers for non-developers to contribute to the Ergo Improvement Proposals process, with Cardano's CIP model cited as a reference. [@CW] scheduled a dedicated EIP community chat for Friday 13 January 2023 at 2 PM UTC. The broader question of how to structure public discourse without creating curation overhead remained open. [@CW, msg#10128, 2023-01-12; @qx(), msg#10173]

### Infrastructure Risks: GitHub & Akka
Two infrastructure dependency concerns were flagged during the month. The Akka library's transition to a closed-source model was noted, with the team still on an old version and no migration path decided. [@Aberg] acknowledged GitHub dependency risk as significant for the ecosystem, noting self-hosted git forge as the only realistic alternative, though no concrete migration plan was proposed. [@Aberg, msg#9261; @kushti, msg#9266, 2022-12-08–09]

## Technical Q&A Worth Preserving

- **Q** (@scalahub, 2022-12-21): Getting a type mismatch error in AppKit 5.0.0 — `ErgoValue[Coll[Byte]]` expects `Coll[scala.Byte]` but receives `Coll[java.lang.Byte]`.
  **A** (@Luivatra): Use the new `ErgoValueBuilder` instead of constructing `ErgoValue` directly. Reference: https://github.com/ergoplatform/ergo-appkit/blob/develop/common/src/main/java/org/ergoplatform/appkit/scalaapi/ErgoValueBuilder.scala

- **Q** (@Luivatra, 2022-12-19): Can order book-style P2S contracts set registers on a box that are only accessed by a second input, not the box itself?
  **A** (@Cheese, msg#9519–9521): Yes. P2S contracts have sufficient flexibility to set registers on a box while only checking a second input's script hash — registers do not need to be accessed solely by the box they are set on. This is precisely why P2S is too complex for standardised message encoding assumptions.

- **Q** (@Luivatra, 2022-12-19): Should message/attachment context variables be standardised, and is P2S the right target?
  **A** (@Cheese, msg#9517): Restricting standardisation to P2PK and multisig outputs makes the most sense. P2S is too flexible; order book contracts and other constructions may use registers in non-standard ways that would break any assumed encoding.

- **Q** (@Luivatra, 2022-12-19): Can new fields be added to outputs without breaking compatibility?
  **A** (@kushti, msg#9564): Depends on the serialisers in use. Some serialisers allow extra bytes, which would permit new field addition via soft-fork.

- **Q** (@scalahub, 2022-12-23): What is the possibility of adding true inclusion height to boxes?
  **A** (Unresolved — filed as GitHub issue #1214): [@scalahub, msg#9789–9790]

- **Q** (@HQΣr, 2022-12-21): How do you fetch Ergo block height from the API and display it in HTML?
  **A** (@Aberg, msg#9642): Use `document.getElementById("id").innerText = data.height` to extract the height property from the JSON response and assign it to the element's inner text.

- **Q** (@HQΣr, 2022-12-22): From which API can current difficulty and total hashrate be fetched?
  **A** (@qx(), msg#9666–9668): Use https://api.ergoplatform.com/info — cache responses hourly to avoid overloading the explorer.

- **Q** (@HQΣr, 2022-12-26 / 2023-01): Can transaction authorisation links (ErgoPaylinks) be created to directly sell tokens from a seller's address, triggering buyer confirmation automatically?
  **A** (No definitive response captured): Flagged as a potential feature need for whitelist token sales and NFT discovery workflows.

- **Q** (@HQΣr, 2023-01): Are there Python smart contracts running on Ergo?
  **A** (@Luivatra, msg#10110): No — ErgoScript is sufficient for contract authoring on Ergo; building a Python-to-ErgoTree compiler would not be a worthwhile effort.
