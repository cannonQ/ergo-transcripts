# Ergo Developer Chat — Monthly Summary
**Month:** 2023-05 | **Chat:** Ergo Developer

---

## Executive Summary

May 2023 was a productive month for Ergo developer tooling, with significant progress across node API improvements, wallet tooling, and dApp infrastructure. Core protocol work centred on EIP-27 pay-2-reemission going live on mainnet and a proposed EIP to simplify fee script logic, while the community saw launches of a community Ergo Explorer, DuckPools smart contract completion, and an ExLE/Nautilus integration. Developer ergonomics around OpenAPI code generation, EIP URI handling, and BIP32 key derivation dominated technical Q&A.

---

## Key Topics This Month

### Node API & Blockchain Indexing
The node API received substantial attention throughout May. Developers encountered persistent issues with OpenAPI code generation — Swagger Codegen was producing broken "AllOf*" classes for Go clients, while OpenAPI Generator with `--skip-validate-spec` produced compilable code. The `/blockchain/balance` endpoint was found to fail on unindexed nodes (indexing offset = 0), returning misleading "Wrong address format" errors compounded by Swagger UI using testnet addresses in examples. Aberg noted that version 5.0.11 would introduce the blockchain index API, and the need for POST endpoints to accept quoted JSON strings for address parameters was raised but remained unresolved by month's end. A separate gap — lack of parameterised address queries with minimum block height filtering — was acknowledged as important but unscheduled.

### EIP URI Fragment Design & Fee Script Proposals
Early in the month, developers debated `#` character handling in the ErgoPayment URI specification, weighing escape sequences against plain text support and backward compatibility with URI RFC standards. Separately, kushti proposed an EIP to set the default transaction fee script to "true" (anyone-can-spend), which would simplify fee collection and allow miners or outsourced agents to collect fees flexibly — this was framed in the context of storage rent collection mechanics, where small Ergo-oriented pools were expected to lead rent collection efforts.

### EIP-27 Pay-2-Reemission
kushti confirmed the active merge of pay-2-reemission boxes with reemission contracts per EIP-27 on mainnet, consolidating miner reward payouts to script-controlled boxes. This was one of the most significant protocol-level announcements of the month.

### Wallet Tooling: Key Recovery, BIP32 Derivation & Satergo
A notable thread resolved Yoroi wallet key recovery by correctly handling a 0-byte prefix on BIP32 private keys — `Arrays.copyOfRange(bip32Data.keyData(), 1, 33)` extracts the 32-byte key, after which an `ExtendedSecretKey` is constructed and the EIP-3 derivation path applied. Aberg developed and planned to publish a recovery tool to GitHub. Related discussions clarified correct EIP-3 derivation path handling in AppKit (`m/44'/429'/0'/0/0`), and confirmed the Satergo built-in node wallet is unsuitable for production use.

### DuckPools & ExLE dApp Progress
DuckPools completed all core smart contracts (partial repayments, liquidation fixes, proxy contract issues resolved) and moved to security audit and testing phases. ExLE achieved a working backend route for Nautilus wallet integration, with main branch merge pending UI completion. These represented two of the most advanced dApp builds active in the channel this month.

### Community Explorer & Tooling Launches
Aco Šmrkas launched the ErgExplorer community alternative to the official Ergo Explorer, featuring transaction views, mobile-responsive design, address/transaction search dropdowns, and htaccess URL rewrite parity with the official explorer. Ergonodes.net also shipped improvements including better node version visualisation and period-based chart filtering. Separately, discussion around automated faucet/airdrop tooling (for presale distribution similar to Hosky or Comet) surfaced, with Pulsarzz building a custom solution.

### Node Version Adoption
Ergo v5.0.10 was released at the start of the month. Satergo's 4-hour auto-update check revealed only ~20 nodes had updated within the observation window, indicating slow adoption among node operators — a recurring concern for protocol upgrades.

## Technical Q&A Worth Preserving

- **Q** (@Aberg, 2023-05-14): How to sign a transaction with a 78-byte BIP32 private key recovered from a decrypted Yoroi wallet?
  **A** (@Aberg, resolved): The 78-byte key includes a 0-byte prefix identifying it as a private key. Solution: `Arrays.copyOfRange(bip32Data.keyData(), 1, 33)` extracts the 32-byte private key. Construct `ExtendedSecretKey(privateKey, chainCode, false, DerivationPath.MasterPath())` — the key is pre-derived to the change level, so only child derivation is needed.

- **Q** (@Aberg, 2023-05-14): How to properly derive EIP-3 addresses using derivation path `m/44'/429'/0'/0/0` with AppKit?
  **A** (@kushti): Create a master key with `DerivationPath.MasterPath()`, then derive: `extMasterKey.derive(DerivationPath.fromEncoded("m/44'/429'/0'/0/0").get()).publicKey().key()`. Do not pass the full derivation path to the `ExtendedSecretKey` constructor — derive incrementally instead.

- **Q** (@Aberg, 2023-05-18): How to generate correct Go client code from the Ergo node OpenAPI spec?
  **A** (@Aberg, resolved): OpenAPI Generator with `--skip-validate-spec` produced compilable code. Swagger Codegen was generating invalid "AllOf*" classes — the Go-lang Swagger Codegen generator is buggy and should be avoided.

- **Q** (various, 2023-05-18): Why does `/blockchain/balance` fail with "Wrong address format"?
  **A** (@qx(), @Aberg): Failures traced to unindexed nodes (indexing offset = 0) and HTTP client quoting of address strings. Nodes indexed at 0/5 behaved differently from 0/10. Swagger UI docs compounded confusion by using testnet addresses in examples.

- **Q** (@kii, 2023-05-21): How to handle third-party transaction fee signing in ErgoScript?
  **A** (@kii): Two patterns: (1) multisig where the third party co-signs alongside the user; (2) a proxy box containing the transaction fee, restricting execution to contracts signed by an authorised user — effectively "pre-authorised fee payment."

- **Q** (@Unknown, 2023-05-08): Are there tools to scan which UTXOs will be available for storage rent collection at a given block height?
  **A**: No longer possible via direct scanning in recent versions due to protocol constraints.

- **Q** (@Glasgow, 2023-05-25): Pay-to-Script (p2s) address implementation examples?
  **A** (@Glasgow): See https://wallet.plutomonkey.com/p2s/ and Scala example at https://scastie.scala-lang.org/FzcCv7pzTb2lQAamWsv1MQ.
