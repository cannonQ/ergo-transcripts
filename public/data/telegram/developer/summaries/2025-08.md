# Ergo Developer Chat — Monthly Summary
**Month:** August 2025 | **Chat:** Ergo Developer

---

## Executive Summary

August 2025 was dominated by the sub-blocks implementation reaching near-complete p2p layer rework, active devnet testing, and the release of Ergo node 6.0.1 alongside Sigma SDK v6.0.2. Alongside core protocol work, several ecosystem projects made meaningful progress: Rosen Bridge advanced Bitcoin Runes integration, Lithos Protocol moved toward testnet with fraud proof contracts, Nautilus wallet gained Keystone hardware wallet integration work, and the Braid whitepaper introduced a novel double merge-mined Bitcoin-Ergo sidechain concept.

---

## Key Topics This Month

### Sub-blocks Implementation
The month's most active engineering thread. kushti advanced the sub-blocks p2p layer rework through multiple iterations: adopting 6-byte weak transaction IDs and high/low bandwidth modes inspired by BIP152 compact block relay, implementing ordering block announcement with input block contents, and testing on a dedicated devnet. By week's end the p2p rework was ~90%+ complete in initial form, with a micronetwork redeployment planned for the start of September following accumulated p2p layer changes. DoS attack vector analysis remained an open concern, and community help was solicited for stress-testing. Four new API endpoints were added to the `/blocks` section (`/bestInputBlock`, `/bestInputChain`, `/{id}/inputBlockTransactions`, `/{id}/inputBlockTransactionIds`) and feedback was requested on their sufficiency. Public testnet deployment with miner support is contingent on 6.0.1 stability and these p2p fixes.

### Ergo Node and Sigma SDK Releases
Ergo node 6.0.1 was released on 2025-08-09, incorporating PR#2235 and PR#2222. Sigma SDK v6.0.2 followed the same week. The sigmastate-interpreter repository adopted the same branching model as the main node repo (master tracks latest release, no develop branch, RC PRs per release). Sigma SDK 6.0.3 candidates were in active review through the month, including PR#1061 (merged) and PR#1069 (NIPoPoWs deserialization, needed for sub-blocks support). A block candidate regeneration test (PR#2237) was added to extend coverage for previous candidate solution acceptance.

### Braid: Double Merge-Mined Bitcoin-Ergo Sidechain
An early draft whitepaper for Braid — a double merge-mined Bitcoin-Ergo sidechain concept — was published by kushti at BetterMoneyLabs. The design leverages Ergo's proof-of-work infrastructure to create a sidechain jointly secured by both Bitcoin and Ergo miners. Community feedback was solicited but no formal responses were recorded in the developer chat during the month.

### Rosen Bridge Development
Rosen Bridge saw sustained delivery across the month: event page UI refinements, Smart Search, sorting, and pagination; Ergo network enhancements for asset handling and box selection; watcher service v2 integrating all chains into a unified scanner; rosen-service refactored with chain manager structure and health-check services. Bitcoin Runes integration reached 4/6 MRs under review with wallet integration (5th MR) in progress. A major bug in the Ergo Multisig package was identified and fixed during Guard-service testing with Rosenet.

### Lithos Protocol
Lithos Protocol made steady progress through August: the difficulty contract was completed, fraud proof contracts reached ~75% completion (~15 of ~25 contracts), off-chain fraud proof code advanced in parallel, and stratum code was optimized and a disconnection bug fixed. The project was preparing for testnet deployment by month end, with documentation improvements and contract cleanup ongoing.

### Nautilus Wallet and Hardware Wallet Integration
Keystone hardware wallet integration with Nautilus reached a functional PR state (opened and in review by anon_br) with parallel firmware work. The integration revealed a significant technical challenge: 161 transient CJS dependencies were doubling the bundle size, requiring lazy loading strategies. Minotaur wallet 3.1.0 was released with asset collapsing, a Buy ERG button, color style updates, and dependency updates. Passwordless transaction signing research continued separately.

### ChainCash Development
ChainCash contract review was completed by kushti with notes to Luca. A refund mechanism for Basis reserve withdrawal was completed (allowing withdrawal 14,400 blocks after an offline tracker announcement). Wallet design drafts were under active discussion in the dedicated ChainCash Telegram group. The USD pool v1 to v2 upgrade path was also under investigation.

### Developer Tooling and Contract Testing
A recurring practical thread through the month concerned best practices for ErgoScript contract debugging. mgPai demonstrated a JSON UTXO conversion method — replacing ErgoTree with a custom compiled version for local evaluation against already-deployed mainnet transactions. Fleet SDK's mock-chain was recommended as the primary local testing tool, though a significant gap was identified: no formal mock-chain documentation exists, flagged as a high-priority need by arobsn. A nested loop stack overflow in ErgoScript (1000×5 iterations) was raised but not resolved.

### Ergo Website Performance
ergoplatform.org received a substantial performance overhaul: Desktop Lighthouse score improved from 36→96, Mobile from 52→81. LCP dropped from 5.6s→0.8s (desktop) and 2.7s→1.1s (mobile). Machine translation rollout and language switcher UX improvements were also completed.

## Technical Q&A Worth Preserving

- **Q** (@Josemi, 2025-08-04): Can a data input reference a box that has already been spent?
  **A** (@kushti): Only within the same block. See [ergoforum.org/t/data-inputs-semantics/654](https://www.ergoforum.org/t/data-inputs-semantics/654). For referencing spent boxes in script, use the certificate pattern by anon_real: [ergoforum.org/t/ergoscript-design-patterns/222/23](https://www.ergoforum.org/t/ergoscript-design-patterns/222/23).

- **Q** (@HQΣr, 2025-08-05): FleetSDK-generated address doesn't match mobile wallet derived addresses.
  **A** (@kushti): Derivation path must be `m/44'/429'/0'/0/0` — note the hardened `0'` after the coin type. The path `m/44'/429'/0/0/0` (without the apostrophe) is incorrect and will produce address mismatches.

- **Q** (@Josemi, 2025-08-05): What happens to an NFT when its box is spent due to storage rent?
  **A** (@Pgr456, @HQΣr): If the box lacks sufficient ERG to cover storage rent, it is collected by the miner. If sufficient ERG remains, the NFT and any tokens persist in the newly recreated box.

- **Q** (@Josemi, 2025-08-20): Best practice for debugging ErgoScript contracts without repeated mainnet interaction?
  **A** (@mgPai): Convert the input UTXO to a JSON string, replace the ErgoTree with a custom compiled version (e.g., removing conditions for isolation testing), and evaluate locally. This works even on already-deployed mainnet transactions and is applicable to other eUTXO chains.

- **Q** (@c8, 2025-08-14): Simple examples for Fleet SDK mock-chain?
  **A** (@anon_br): Simplest example in the fleet-by-example package: https://github.com/fleet-sdk/fleet-by-example/blob/master/tests/timedFundContract.spec.ts

- **Q** (@Josemi, 2025-08-19): "Maximum call stack size exceeded" error when executing ErgoScript with double nested loops at 1000×5 iterations (works at 100).
  **A** (@anon_br): Clarification requested — is this running on mock-chain? *(Unresolved: Josemi confirmed mock-chain use but no root cause identified.)*

- **Q** (@Pgr456, 2025-08-22): Is it possible to access current chain parameters like `minValuePerByte` in ErgoScript?
  **A** (@kushti): No direct access exists. However, a trustless oracle is possible: every 1,024 blocks (one voting epoch), parameters are written to the block extension section. A contract can update parameters via Merkle proof validation against such block headers.

- **Q** (@Patato, 2025-08-22): Ergo AppKit throws `IllegalArgumentException: Network type of NodeInfo does not match` when connecting to a DevNet node.
  **A** (@kushti): Root cause is that DevNet nodes report their network type as `devnet` while AppKit only accepts `mainnet` or `testnet`. Fix: next DevNet deployment will have the node report as `testnet`. Interim workaround: use the TESTNET enum in AppKit.

- **Q** (@Patato, 2025-08-22): Need stress test scripts for DevNet transaction generation.
  **A** (@kushti): Use wallet REST API (`/wallet/payment/send`) via shell scripts — simpler and more practical than AppKit. Unlock wallet first, then loop `send.sh` with 1-second intervals using `curl` POST.

- **Q** (@Josemi, 2025-08-19): How to add a token logo to Nautilus wallet?
  **A** (@Patato, community): Add a `.png` file to `/public/icons/assets` and map the token ID in `src/mappers/assetIconMap.ts`.

- **Q** (@Aco Šmrkas, 2025-08-22): Is there API documentation for sub-blocks?
  **A** (@kushti): Four new API methods added to the `/blocks` section:
  - `GET /blocks/bestInputBlock` — returns best ordering and input block IDs
  - `GET /blocks/bestInputChain` — returns array of best input block chain IDs
  - `GET /blocks/{id}/inputBlockTransactions` — returns input block transactions
  - `GET /blocks/{id}/inputBlockTransactionIds` — returns input block transaction IDs
