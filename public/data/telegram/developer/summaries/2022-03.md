# Ergo Developer Chat — Monthly Summary
**Month:** 2022-03 (covering through 2022-04-17)
**Chat:** Ergo Developer

---

## Executive Summary

March 2022 developer activity centred on low-level protocol work: diagnosing a network-wide node synchronisation stall tied to the emission decrease, reverse-engineering and documenting the underdocumented P2P protocol, and clarifying transaction throughput limits ahead of the v5.0 JIT compilation upgrade. A secondary thread ran through Rosen Bridge governance design, specifically the staking and slashing model required to align guard set incentives.

---

## Key Topics This Month

### Node Synchronisation Stall (Late March 2022)
Multiple nodes experienced synchronisation failures lasting 30+ minutes, traced to elevated memory requirements in blocks immediately preceding the emission decrease to 63 ERG. NBMiner did not account for these demands. The root cause at the script or storage level was not fully resolved, and no post-mortem was shared publicly during the period.

### P2P Protocol Documentation
The Ergo node uses a custom TCP-based protocol that was minimally documented in official sources. Aberg (Satergo dev) undertook a systematic reverse-engineering effort from the Scala source, uncovering several legacy encoding anomalies: the address length field includes an extra 4 bytes (a historical mistake that cannot be corrected without breaking backward compatibility), and the port is encoded as a 4-byte `Int` rather than a 2-byte `UShort`. A pull request was submitted to `ergodocs` adding a comprehensive P2P message specification with VLQ encoding reference.

### Transaction Throughput and JIT Compilation
A developer attempting to submit 20 chained transactions observed that only 3 confirmed per block. The block computational cost cap of 8M (miner-votable) was identified as the limiting factor. JIT compilation, anticipated in v5.0, is expected to substantially reduce per-script execution cost and increase effective block throughput.

### Rosen Bridge Guard Set Governance
Discussion clarified that Rosen Bridge guard set participants must be known ecosystem actors with established reputations. The security model relies not only on the basic action flow but on staking and slashing incentives to deter malicious behaviour. No implementation specification was shared beyond the governance principle.

### Node Seeding and Bootstrap Centralisation
The removal of IRC-based peer seeding in earlier versions left Ergo without a DNS seed or DHT equivalent. Current peer discovery depends on a small set of known seed peers, creating a potential bootstrap vulnerability if those peers are DDoS'd. No remediation was proposed during the period.

## Technical Q&A Worth Preserving

- **Q** (@Aberg Satergo dev, 2022-04-13): Where is Ergo's equivalent of Bitcoin's `getaddr` message documented in the node source?
  **A** (@Aberg Satergo dev): Not documented externally. After reading the Scala source directly, the implementer found the connection closes after mutual handshake exchange. A separate bug was also identified: a try-with-resources block was prematurely closing the socket input stream when closing the `DataInputStream`.

- **Q** (@Aberg Satergo dev, 2022-04-13): Why does the node add 4 bytes to the address length in the P2P handshake?
  **A** (@Aberg Satergo dev): Legacy encoding error — the port byte length was inadvertently added to the IP address length field. Cannot be corrected now due to backward compatibility requirements.

- **Q** (@Luivatra, 2022-04-15): Is there a per-address per-block transaction confirmation limit? 20 chained transactions were submitted but only 3 confirmed per block.
  **A** (@qx()): The block computational cost limit is 8M (miner-votable). Chaining 20 transactions likely exceeds this unless each transaction carries a very low script cost. JIT compilation in v5.0 is expected to significantly reduce per-script overhead.

- **Q** (@Aberg Satergo dev, 2022-04-15): Is node seeding only done via known hardcoded peers? Doesn't this create a centralisation risk if seed peers are DDoS'd?
  **A** (@Aberg Satergo dev): IRC-based seeding existed in old versions but was removed due to dependency on external IRC server availability. No DNS seed or DHT equivalent is currently implemented for Ergo.
