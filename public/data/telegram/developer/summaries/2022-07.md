# Ergo Developer Chat — Monthly Summary
**Chat:** Ergo Developer | **Month:** 2022-07

---

## Executive Summary

July 2022 was dominated by node stability and infrastructure concerns, particularly around the v4.0.35 peer management regression and AVL tree persistence failures during upgrades, alongside the launch of the 5.0 testnet. A critical sigma-rust token limit bug (255 tokens per box) that caused significant user funds loss drove important cross-wallet compatibility discussions, while the month closed with early-stage exploration of market-driven emission contracts, optimistic rollups, and decentralized grid trading.

---

## Key Topics This Month

### Node Stability: v4.0.34 AVL Tree Failures & v4.0.35 Peer Management Regression
Two distinct but related node stability issues dominated July. Upgrading from v4.0.20.1 to v4.0.34 triggered `PersistentBatchAVLProver` assertion failures resulting in `fullHeight: null` and inability to sync, with suspected database corruption. Shortly after, v4.0.35 introduced aggressively tuned peer removal on delivery failures, causing nodes to drop from 100+ peers to fewer than 10. Multiple operators downgraded to v4.0.34 as a workaround. Neither root cause was fully resolved within the month.

### 5.0 Testnet Launch & Block Parameter Experiments
The 5.0 testnet officially launched in late July with details posted to the Ergo Forum. Earlier in the month, discussion focused on 5-second block time experiments, including the 720-block cooldown period and its downstream effects on script recognition, fee mechanisms, and any dApps relying on block height rather than timestamps. kushti noted a dedicated testnet channel exists for parameter-tweaking experiments and invited contributors who don't necessarily need to write code.

### sigma-rust Token Limit Bug & Wallet-Level Responsibility
A maximum of 255 tokens per box — enforced in sigma-rust — caused a user to lose approximately $30k. Discussion examined whether the fix belonged at the protocol level, the wallet level, or the transaction builder level. Satergo and Ergo Android wallet were unaffected, as they do not rely on sigma-rust for this logic. The broader question of how responsibility is distributed for bugs in upstream libraries (sigma-rust) versus downstream wallet implementations remained unresolved.

### Cloudflare Blocking Non-Browser API Access
The Ergo Explorer API at ergoplatform.com was returning HTML browser-verification responses instead of JSON, blocking programmatic clients including Tor users and wallet software like Satergo. kushti disabled verification for Tor traffic as an interim fix. This highlighted the fragility of infrastructure dependencies on third-party DDoS mitigation services for developer tooling.

### Node Logging & Terminal-less Process Execution
Running the Ergo node from a terminal-less Java process (e.g., via `java.lang.Process`) caused STDOUT/logging to be lost after the initial INFO line due to logger reconfiguration during startup. A workaround using the `pty4j` pseudo-terminal library was confirmed effective. The underlying logger reconfiguration sequence was flagged for investigation but not resolved.

### Market-Driven Emission, Optimistic Rollups & Grid Trading
kushti posted forum sketches for three forward-looking design concepts: market-driven emission contracts, optimistic rollup designs using Merkle trees, and decentralized grid trading on Ergo. None were elaborated in chat; all remained at the forum-sketch stage entering August.

## Technical Q&A Worth Preserving

- **Q** (@Aberg, 2022-07-18): Node crashes with `PersistentBatchAVLProver` assertion failure after upgrading from v4.0.20.1 to v4.0.34, even after deleting blockchain data. `fullHeight` reports null and sync cannot complete.
  **A** (@kushti): Requested full error context; suspected database corruption; committed to investigation. Root cause not confirmed within the month.

- **Q** (@Aberg, 2022-07-25): Why is the Ergo node not runnable from a terminal-less Java process? Running via `java.lang.Process` causes logging output to disappear after the first INFO line.
  **A** (@kushti): After the initial INFO log line, the logger is being reconfigured during startup; the logging configuration sequence needs investigation.
  **Follow-up** (@Aberg): Confirmed workaround — using the `pty4j` pseudo-terminal library and piping `ptyProcess.getInputStream().transferTo(System.out)` restores STDOUT correctly.

- **Q** (@Flying Pig, 2022-07-28): v4.0.35 node dropping peers rapidly; public node down from 100+ to ~30 connections. Reproducible across multiple nodes.
  **A** (@Luivatra): v4.0.35 likely introduced more aggressive peer removal on delivery failure.
  **A** (@Flying Pig): Confirmed — downgrading to v4.0.34 restored normal peer counts immediately. Issue appears to be an unintended consequence of the peer management change.

- **Q** (@Aberg, 2022-07-09): Is the 255-token-per-box limit stored as a `short` per the EIP spec?
  **A** (@Luivatra): Confirmed in the spec comment.

- **Q** (@MrStahlfelge, 2022-07-09): Should wallet developers feel responsible for users affected by sigma-rust bugs, given sigma-rust is an upstream library?
  **A** (@Aberg): Responsibility is unclear for bugs in upstream libraries. Satergo implements logic per node source code; the developer does not own the inner workings of sigma-rust box handling.

- **Q** (@Aberg, 2022-07-28): Why do many nodes report 169.254.x.x IP addresses in peer discovery when `declaredAddress` is not configured?
  **A** (@Aberg): These are IANA-reserved link-local addresses; flagged as a potential bug in peer discovery and reporting logic when no declared address is set.

- **Q** (@Aberg, 2022-07-30): Testnet faucet claims a brand-new, never-used address has already received ERG.
  **A** (@Moein Zargarzadeh): Requested the address for investigation; resolved manually.

- **Q** (@scalahub, 2022-07-30): What is the default oracle participant count in Oracle Pool v2.0?
  **A** (@scalahub): Default is 4, though it depends on the specific use case configuration.

## Open Questions Entering Next Month

- **sigma-rust token limit fix**: No consensus reached on whether fixing the 255-token-per-box limit belongs at the protocol level, the transaction builder level, or the wallet level. Security and backwards compatibility trade-offs unresolved.
- **v4.0.35 peer management**: The aggressive peer removal behavior in v4.0.35 was worked around via downgrade but not formally root-caused or patched. Whether the behavior is intentional or a regression remains open.
- **AVL tree persistence failures**: The `PersistentBatchAVLProver` crash on upgrade from v4.0.20.1 to v4.0.34 was flagged by kushti but not conclusively diagnosed.
- **BIP32 standard derivation flag PR**: Flagged by Aberg as having been pending merge for months with no explanation; blocking status unresolved.
- **Logger reconfiguration in terminal-less processes**: Workaround (pty4j) identified but underlying logger reconfiguration sequence in startup not fixed.
- **Flux infrastructure integration**: Armeanio's Flux onboarding proposal received no formal decision; risk and performance assessment pending.
- **Forum sketches for emission contracts, rollups, grid trading**: All three kushti forum posts were at sketch stage only; no implementation timelines or formal EIPs proposed.
