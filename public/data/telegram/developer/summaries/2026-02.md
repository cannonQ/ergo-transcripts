# Ergo Developer Chat — Monthly Summary: February 2026

---

## Executive Summary

February 2026 was dominated by sub-blocks devnet testing and the Ergo 6.0.x release pipeline, with kushti driving iterative protocol improvements alongside community tooling contributions. Secondary themes included developer experience improvements (Schnorr signature endpoints, Fleet SDK debugging, ErgoTree utilities) and privacy/infrastructure concerns surfacing through NFT viewer design and Spectrum DEX backend outages.

---

## Key Topics This Month

### Sub-blocks Devnet & Protocol Development

The sub-blocks devnet was relaunched during the first week of February with a 60-second average ordering block delay and 60 input blocks per ordering block, currently running with 3–4 mining peers. Code improvements and documentation updates were ongoing in parallel with the release pipeline. Discussion touched on the broader question of whether microblocks, sidechains, or on-chain optimisations represent the right path for faster confirmations, with rough consensus that a concrete set of deployed application patterns (Oracle Pools, Rosen Bridge, Spectrum, AMM pool contracts, Trade House, lending contracts) now makes it feasible to optimise meaningfully — unlike earlier in Ergo's history when chains were relatively empty.

### Ergo 6.0.x Release Pipeline

PRs #2252 and #2290 were identified as ready for community review, targeting 6.0.3 and 6.0.4 respectively. The plan is to extract sub-blocks-related PRs one-by-one after these releases land, allowing incremental review. A new `/utils/schnorrSign` endpoint (PR #2290) was added to enable Schnorr signature generation for arbitrary messages using node wallet keys — directly useful for off-chain applications interacting with Schnorr signature verification contracts.

### Developer Tooling & SDK Issues

Fleet SDK input selection behaviour caused confusion when UTXO register values (R5) appeared to differ between local representation and node validation. The root cause was Fleet's default prioritisation of older inputs; the fix is to use `ensureInclusion: true` for contract inputs. Separately, a complex transaction signing failure in Nautilus ("invalid public key") was investigated but not fully resolved — suspected to relate to Nautilus secret/input selection logic. ErgoTree template hash generation was also discussed; no dedicated node endpoint exists, but community tools at `ergolabs.degens.world/tools` and cannon_q's Python scripts serve as working references.

### NFT Viewer & Privacy

A new NFT viewer implementation (PR #2288) was flagged for exposing node IP addresses when rendering external URLs (IPFS or otherwise) from NFT R9 registers. The feature was confirmed complete but will be split from the dark mode PR and reviewed independently, with privacy implications under active consideration before any merge.

### Community Infrastructure & Tooling

cannon_q shipped a live storage rent demurrage tracker at `srtracker.degens.world`, cataloguing 441K+ storage rent claims with a mobile-friendly interface updated every 15 minutes. The SigmaVerse repository was moved to the Degens-World GitHub organisation with contributor write access opened for dApp listings. The Ergo Book project began conversion to mdbook format, with a Reddit AMA planned featuring Grayman, Armeanio, and kushti. Patato published a fork of the Ergo Paper Wallet with BIP39 passphrase support, verified against Satergo, with all UI languages updated.

### Spectrum DEX Backend Outages

Both `dex-api.sigmaspace.io` and `api.spectrum.fi` went down due to API cache exceeding available RAM. The sigmaspace instance was restored with monitoring added; the episode highlighted the need for health check API methods across Ergo's DeFi infrastructure.

## Technical Q&A Worth Preserving

- **Q** (@Josemi, 2026-02-05): R5 register values differ between Fleet SDK local representation and node validation — same UTXO shows correctly locally but node reports different values in diagnostic info. What's happening?
  **A** (@anon_br, 2026-02-05): Input selection is the issue. Fleet prioritises older inputs by default. Use `.from(parseBox(game.box), { ensureInclusion: true })` to enforce the intended contract input. Alternatively, enable Developer Mode in Nautilus to inspect the exact signing JSON being submitted.

- **Q** (@Pulsarz, 2026-02-07): How do you convert an ErgoTree to a template hash?
  **A** (@kushti, 2026-02-07): No dedicated node endpoint is needed — retrieve the tree from P2S and strip constants to generate the template. cannon_q's Python scripts (shared in recent weeks) serve as a working reference. Community tools are also available at `ergolabs.degens.world/tools`.

- **Q** (@HQΣr, 2026-02-07): Transactions are being submitted and returning TXIDs but not appearing in Ergo Explorer for hours — what's going on?
  **A** (@kushti, 2026-02-07): Likely a double-spending scenario where a competing transaction with a higher fee is already in the mempool. Resolution: submit directly via node endpoints rather than the Explorer submit endpoint.

- **Q** (@Netim, 2026-02-05): Node stuck at height 5812 after database reset and re-sync — how to diagnose?
  **A** (@kushti, 2026-02-05): Logs needed for diagnosis; suspected invalid input block header generation on the 213 node. Logs were subsequently provided via Google Drive for investigation.

- **Q** (@Moein Zargarzadeh, 2026-02-07): Both Spectrum backend APIs are down — `dex-api.sigmaspace.io` and `api.spectrum.fi`.
  **A** (@Pgr456, 2026-02-07): API cache exceeded available RAM. sigmaspace instance restored and monitoring added. Health check API methods identified as needed for ongoing infrastructure resilience.

## Open Questions Entering Next Month

- **NFT Viewer IP exposure**: Rendering external URLs from NFT R9 registers leaks node IP. Privacy mitigation strategy and merge criteria remain undefined.
- **Terminus ErgoAuth "not on host" error**: Persistent on iOS despite matching hostnames in ergoauth:// URL and replyToUrl; desktop Nautilus works correctly. Root cause and sigmaBoolean format requirements unclear.
- **Nautilus complex transaction signing failure**: "Invalid public key" error on multi-input transactions not reproducible via node endpoint. Likely in Nautilus secret/input selection logic; not yet resolved.
- **Sub-blocks path forward**: Broader architectural question — microblocks vs. sidechains vs. on-chain optimisations — deferred pending 6.0.3/6.0.4 releases and community discussion post-Ergoversary.
- **SigmaVerse deployment process**: Contributor workflow (GitHub-based vs. manual additions) needs clarification before full contributor access is formalised.
- **Dexy Gold Oracle Pool v2 / GORT rewards**: V2 mechanics mentioned but not fully specified; details outstanding.
