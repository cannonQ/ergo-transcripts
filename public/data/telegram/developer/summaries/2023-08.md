# Ergo Developer Chat — Monthly Summary
**Month:** August 2023
**Source chats:** W32, W34, W35

---

## Executive Summary

August 2023 was dominated by foundational developer tooling improvements, P2P library development, and the announcement of ErgoHack VII. Significant technical discussions centred on block time optimisation tradeoffs, API infrastructure limitations (timeout issues, JSON streaming format inconsistencies, mempool-awareness gaps), and the challenges of supporting Python developers with incomplete tooling. The month closed with growing ecosystem momentum around Rosen Bridge cross-chain deployment, Paideia DAO public beta, and the ErgoScript compiler separation enabling lighter-weight application development.

---

## Key Topics This Month

### P2P Protocol & Ergonnection Library Development
@Aberg iterated on the Ergonnection P2P library throughout the month, addressing connection handling edge cases. The core question was how to handle invalid magic bytes received during P2P communication — consensus with @kushti was that connections should be closed and peers banned, mirroring existing node behaviour. A subtle root cause was identified: empty magic byte arrays arise from stream EOF/closure conditions, not protocol violations, because `readNBytes()` returns a shorter array on EOF rather than throwing an exception. The library received fixes for testnet magic constants and improved disconnection handling, with a Maven Central release in preparation.

### Block Time Optimisation: Tradeoffs and Dependencies
A sustained technical discussion examined the feasibility of increasing chain speed. Key complications identified: contracts relying on block height rather than timestamps would behave differently at faster block times; emission contracts would tail off faster; and security considerations require worst-case block time assumptions to be respected as part of Ergo's design ethos. @kushti noted a testnet environment exists for parameter experimentation but that no champion has yet stepped up to lead the study. The conclusion was that a concrete set of deployed applications (Rosen Bridge, Oracle Pools, Spectrum DEX, GARD protocols, Trade House, lending contracts) now provides a realistic benchmark for optimisation — unlike earlier in Ergo's history when chains were empty.

### API Infrastructure Gaps
Multiple API-related issues surfaced across the month. The `/api/v1/boxes/unspent/byTokenId/` endpoint on the Cornell node was experiencing timeouts. A consistent discrepancy of 3 transactions per day was observed between ergoplatform.com's explorer and the Aneta BTC and Cornell explorers (which matched each other). The JSON streaming endpoint for tokens returns objects without array brackets or comma separation, making parsing awkward; debate arose over whether to fix the format (a breaking change) or introduce a versioned v2 endpoint. There is also no official endpoint combining unspent box filtering by token ID with mempool visibility, creating an infrastructure gap for dApp developers.

### ErgoScript Compiler Separation
The Sigma repository's develop branch now separates the ErgoScript compiler from the full protocol implementation. This significantly reduces the footprint for Java/JVM applications that only need compilation without the full sigmastate (reducing dependency from ~8 MB). This was highlighted as particularly relevant for lighter tooling and mobile development contexts.

### Python Developer Tooling & ErgPy
Growing interest from Python developers revealed gaps in the ErgPy library, particularly around UTXO discovery and box selection. One developer encountered a bug where ErgPy would miss boxes during token distribution, showing a lower balance than actually available — the workaround was manual UTXO consolidation before running distribution scripts. The JDE transaction builder was confirmed as archived and no longer compiling. @qx() shared a Python file for airdrop use as a copy-paste reference. Broader discussion acknowledged that Python tooling remains incomplete compared to the JVM and JavaScript/TypeScript ecosystem.

### ErgoHack VII & Ecosystem Milestones
@kushti officially announced ErgoHack VII for October 13–15, 2023, with an 18K SigmaUSD prize pool centred on "The Future of Finance and the Path to Adoption," immediately followed by an Ergo Summit. Simultaneously, the GreasyCEX Greasy Royale Mission #2 targeting a MEXC listing was confirmed with ~200 NFTs and 100% reward distribution to donors (Mission #1 had raised 2,464 ERG). On the ecosystem side, Rosen Bridge cross-chain deployment to Cardano was described as imminent, Paideia DAO public beta was weeks away, and Spectrum Finance had expanded to Cardano with a cross-chain swap roadmap.

### Fee Markets & Spam Protection
Discussion of spam resistance drew out several underappreciated features: nodes have configurable minimum propagation fee rates to prevent low-fee transactions from entering the mempool; a fee estimation API exists in the node but is untested and unused by applications; and transactions left unpicked in the mempool eventually flush out after enough blocks pass and higher-fee transactions fill blocks. @kushti referenced the 2020 autumn spam events as historical precedent for fee market self-correction.

### Rosen Bridge Watcher Updates
Watcher operators were advised to update to version 3.1.1 (a 3.0.1 release had contained a bug that required holding off). Watcher UI version 2.2.1 is the current release, updatable via `docker compose pull && docker compose down && docker compose up -d`. A manual TX was submitted to GARD's DAO to refund users whose permits were slashed from an accidental high-fee transaction that failed the bridge due to a missed signature; resolution was actively being worked on.

## Technical Q&A Worth Preserving

- **Q** (@Aberg, 2023-08-07): When a wrong magic number is received in P2P communication, should the connection be closed?
  **A** (@kushti): Yes — close the connection. The node bans the peer for that behaviour.

- **Q** (@Aberg, 2023-08-07): Is there any valid scenario where a wrong magic number is received?
  **A** (@kushti): Possibly if connecting to a testnet node; otherwise this should not occur under normal operation.

- **Q** (@Aberg, 2023-08-07): How can the magic number at the start of a message be empty?
  **A** (@Aberg, self-resolved): Stream closure or EOF. The `readNBytes()` method returns a shorter array on EOF without throwing an exception. Four empty bytes means no message arrived at all — not a protocol violation.

- **Q** (@Luivatra, 2023-08-28): What timestamp value would be used in the proposed validation context feature?
  **A** (@kushti): Initially proposed `now()`, reconsidered to the last header's timestamp to maintain consistency with current behaviour. This would also facilitate storage rent transaction creation.

- **Q** (@Pulsarz, 2023-08-22): Is there an endpoint returning unspent boxes by token ID with mempool visibility?
  **A** (community): No. Current workaround requires manual mempool extraction. A combined token-filtered unspent box endpoint with mempool integration would significantly reduce explorer infrastructure burden — flagged as a needed addition.

- **Q** (@qx(), 2023-08-27): The `/api/v1/boxes/unspent/byTokenId/` endpoint on the Cornell node is timing out. Is this a framework default, server configuration, or data volume issue?
  **A** (unresolved): Confirmed timing out on both browser HTTPS and direct HTTP. Root cause under investigation.

- **Q** (@qx(), 2023-08-28): Why does ergoplatform.com's explorer show 3 more transactions per day than the Aneta BTC and Cornell explorers, which match each other?
  **A** (unresolved): Consistent 3-TX discrepancy confirmed across days; root cause not identified.

- **Q** (@Aberg, 2023-08-31): The JSON streaming API endpoint returns objects without array brackets or comma separation — is this intentional?
  **A** (@Pulsarz): The stream format is intentional but acknowledged as annoying to parse. (@Aberg): Argued it should be fixed regardless of backward compatibility. (@Luivatra): Suggested a versioned v2 endpoint as the path forward. No decision recorded.

- **Q** (community, 2023-08-28): Can bulletproofs be verified in ErgoScript?
  **A** (@kushti): Bulletproofs are a zero-knowledge technique enabling range proofs and generic statement proofs without a trusted setup — essentially a specific form of Sigma protocol. If verification can be done in ErgoScript, it would open a wide class of zero-knowledge protocols usable alongside Ergo contracts, including Groth one-out-of-many proofs (used in early Zcoin) and verifiable credentials. Sensitive data could be verified on-chain without disclosure.

- **Q** (@Flying Pig, 2023-09-01): The JDE transaction builder is archived and won't compile. Is there an updated Python TX builder?
  **A** (@Glasgow): ErgPy with examples is the current recommendation. (@qx()): A Python airdrop file is available as a copy-paste reference.

- **Q** (@HQΣr, 2023-08-18): Spanish community requests fees be deducted from the amount sent rather than paid separately.
  **A** (@Pulsarz, @qx()): Rejected as poor UX. If sending 100 ERG, the receiver must receive 100 ERG. Variable fees make deduction-from-amount impractical.

- **Q** (community, Ergoversary AMA): Can Ergo suffer the same DDoS/spam attack seen on other chains?
  **A** (@kushti): Ergo has configurable minimum propagation fee rates at the node level, preventing low-fee transactions from entering the mempool. A fee estimation API exists in the node (untested, unused by applications). Transactions that remain unpicked eventually flush from the mempool after sufficient blocks pass. Historical spam events in autumn 2020 resolved via fee market self-correction. Users can also manually set higher fees via the slider in Nautilus to get picked up ahead of spam.
