# Ergo Developer Chat — Monthly Summary
**Month:** September 2024

---

## Executive Summary

September 2024 was dominated by two converging development tracks: the Ergo 6.0 protocol soft-fork (culminating in the `v6.0.0-alpha1` DevNet release) and the ongoing sub-blocks/input-blocks implementation targeting throughput improvements. Alongside core protocol work, significant ecosystem tooling matured across wallets (Nautilus v0.14.0), bridges (Rosen Bridge watcher releases), SDKs (Fleet SDK v0.7.0), and explorer infrastructure (sigmaspace.io), while ErgoHack IX was announced with a UI/UX and mining focus for late October.

---

## Key Topics This Month

### Ergo 6.0 Protocol Development
The month saw steady progress toward Sigma 6.0, with multiple PRs merged into the candidate branch covering new numeric methods, unsigned `BigInt` support, higher-order lambdas, and new collection methods. `v6.0.0-alpha1` was released for DevNets on 2024-09-19, featuring `Box.getReg` with computable indices, higher-order functions, and unsigned big integers. Sigma-Rust 3.0.0 was also published during the month, and bounties were added to Sigma-Rust 6.0 issues to attract contributors. The next DevNet release was anticipated within 7–10 days of week 39, adding `Some`/`None` constructors and revised AVL+ tree operation semantics.

### Sub-blocks / Input Blocks Implementation
kushti progressed the sub-blocks protocol throughout the month, implementing P2P message processing (sub-block transmission, transaction requests, delivery), Stratum proxy modifications, and candidate generation work. The design is inspired by the Prism protocol but grounded in GKL-15-based formal security proofs. A dedicated DevNet for throughput testing was described as imminent by late September. Load testing assistance and Stratum proxy modification help were solicited from the community. A parallel thread cautioned against inflated TPS claims from other ecosystems (e.g., 40k–70k figures from Prism and similar chains) due to reliance on datacenter testbeds, empty state assumptions, and hybrid network conditions.

### Node Releases & Infrastructure
`Ergo 5.0.23` was released after a threading issue in the indexer block-caching layer was identified and resolved. A disk space exhaustion incident on `api.ergoplatform.org` (PostgreSQL fill) was fixed promptly. `5.0.24` was in candidate testing by month end. A bounty-attached GitHub issue (#2184) was opened for community contributors. The maximum wallet input limit (default 100 boxes) was clarified as a wallet-level configuration, not a hard protocol constraint.

### Rosen Bridge Watcher Releases
Watcher-app v2.3.0 and watcher-service v3.2.1 shipped with eRSN token support, message retry logic, and improved health monitoring. A final round of testing requiring 30+ volunteers was announced in week 39, with a private testbed made available. sigmaspace.io added Rosen Bridge watcher endpoints to its explorer API, now sponsored by the Ergo Foundation for server costs.

### Nautilus Wallet & Fleet SDK
Nautilus v0.14.0 added a Transaction History view, set a 1,000 USD minimum liquidity threshold, switched to sigmaspace.io as the default explorer backend, and fixed non-standard token minting register parsing (v0.14.1). Fleet SDK v0.7.0 introduced an `ensureInclusion` flag for input ordering guarantees, a breaking change to the `decode` function returning `SConstant` objects, and improved Babel Box validation. Blockchain provider retry logic and UTXO diff calculation fixes for multi-token scenarios were included in v0.6.4 earlier in the month.

### sigmaspace.io Explorer & Storage Rent Dashboard
pgr456 launched a live storage rent tracking dashboard (sigmaspace.io/en/storage-rent) indexing the full chain in under 24 hours, surfacing upcoming rent-liable boxes and 24h/7d/30d statistics. An independent chain indexer and REST API compatible with `ergoplatform.org` endpoints was added to the roadmap. Three API endpoints were documented at `api.sigmaspace.io/docs` and opened for community testing. The Ergo Foundation agreed to sponsor server costs.

### ChainCash Development
dusek identified and fixed incorrectly used testnet oracle and buyback NFT IDs in mainnet contracts. PRs for note indexing by holder public key (#55) and note redemption (#56) were reviewed and approved. A timelock redemption feature was identified as needed and added to the work backlog.

### Dexy Protocol
All mainnet tokens were issued across the month. Payout contract audit results were received in week 39, flagging issues for remediation. Community discussion clarified the token issuance model: the preservation law applies only to new tokens (where token ID equals the first input box ID), enabling flexible multi-output distribution without requiring a single "issuance output." A proposal to enforce stricter EIP-4 compliance for consistent indexing was discussed but not formally decided.

### Paideia DAO
Context variable count was reduced from 5+ to 3 for affected contracts. Off-chain code was updated and a final testing round was deploying by the end of the month. The Paideia platform hosted a Cornell AAP student DAO with token staking, digital economy features (printing/plotting payments, tool trading, fines, p2p transfers), and annual reset as cohorts graduate. A serialization ordering discrepancy between Scala and sigma-rust implementations was investigated, with a proposed workaround of passing pre-serialized transaction bytes to Nautilus for signing.

### ErgoHack IX
ErgoHack IX was announced for October 25–31, 2024, themed around UI/UX Design and Mining Solutions. Nine community-submitted ideas were crowdsourced: limit orders for Spectrum DEX, mempool visualization, a routing layer, a token transaction explorer, SigmaFi UI improvements, procedural NFTs, a FIMO launch tool, anonymous voting via stealth addresses, and a user-friendly L2 wallet. A tenth idea—completing the new ergoplatform.org website design—was proposed. Projects already under active development were declared ineligible.

### IPFS Gateway Migration
Cloudflare's IPFS gateway deprecation (effective August 14, 2024) was flagged as requiring action from NFT platforms including SkyHarbor, ErgExplorer, and ErgTokens to migrate image URL resolution to `ipfs.io` or `dweb.link`.

### Celaut Project
The reputation system for Celaut nodes was completed and the payment system entered active development. Ergo node configuration options were added (local wallet mnemonic, cold wallet, node URL). Work began on a Benefactor Platform to allow other projects to use the CAT benefaction system.

## Technical Q&A Worth Preserving

- **Q** (@nicolavon, 2024-09-02): Why does `/wallet/transaction/generate` return a 400 error with "Validation failed on ValidationRule(1012, For version greater than 0, size bit should be set)"?
  **A** (@kushti): `inputsRaw` must contain binary-encoded box bytes obtained via `/utxo/byIdBinary/{boxId}`, not box IDs. `dataInputsRaw` can be omitted. Fee must be ≥1,000,000 nanoERG.

- **Q** (@qx(), 2024-09-03): Wallet scan not initiating on testnet despite full node sync, index sync, and wallet initialisation all completing.
  **A** (@qx()): Resolved by: (1) syncing node without index fully; (2) initialising/restoring wallet to trigger scan or manually triggering scan from Swagger; (3) shutting down and enabling `extraIndex` in config; (4) deleting `/history/extra` folder; (5) restarting node.

- **Q** (@Michael, 2024-09-09): Node wallet failing with `MaxInputsExceededError` when attempting to send more than 100 UTXOs in a single transaction.
  **A** (@Pulsarz): The 100-box limit is a wallet-level configuration, not a hard protocol restriction. Configurable via `ergo.conf` at `application.conf#L309`.

- **Q** (@nicolavon, 2024-09-12): How to securely construct ECDSA signatures locally without passing a private key to `/wallet/transaction/sign`?
  **A** (@kushti): Use external libraries for local signing: ErgoAppKit (Java/JVM), sigma-rust (Rust + bindings), Fleet SDK (JS/TS), or ergo-wallet (Java).

- **Q** (@mgpai, 2024-09-17): Is it possible to mint a 2-unit token with one unit in the issuance output and one unit in another output?
  **A** (@kushti): Yes. No single "issuance output" is required. Only the preservation law applies (sum in outputs ≤ sum in inputs), and it is waived for new tokens where the token ID equals the first input box ID.

- **Q** (@Aberg, 2024-09-17): Why does `DerivationPath.fromEncoded("m/44'/429'/0'")` return a path with an initial 0 in `decodedPath`, breaking equality checks with paths created from integers?
  **A** (@kushti): The constructor taking a Scala `IndexedSeq[Int]` includes 0, following BIP-32 where `DerivationPath(List(0), publicBranch = false)` corresponds to the master key, with `44'/429'/0'` derived from it.

- **Q** (@Pgr456, 2024-09-25): How to go from ErgoTree to a template? Where is the documentation on constants position/index?
  **A** (@luivatra): You go from an EIP-5 script → template (exported to JSON) → tree by applying parameters. Constant indices are visible in the JSON. Example: [ActionSendFundsBasic.json](https://github.com/paideiadao/paideia-sdk/blob/dev/src/main/resources/ergoscript/ActionSendFundsBasic/1.0.0/ActionSendFundsBasic.json).

- **Q** (@Aberg, 2024-09-06): Is there a method/API to convert bytes to an AST representation of ErgoScript?
  **A** (@kushti): Use `ErgoTreePredef` to construct and `toProposition(true)` to render as AST-like code. Pretty-printing PRs #812 and #831 exist but are incomplete; they would improve readability when finished.

- **Q** (@dusek_, 2024-09-28): Could Nexa's efficient block propagation (transmitting transaction diffs instead of full transactions) be useful for Ergo?
  **A** (@kushti): This is already done in Bitcoin (compact blocks) and is part of Ergo's sub-blocks implementation.

- **Q** (@ioannisleontiadis, 2024-09-28): Is it feasible to build a tutoring platform where Ergo funds release in chunks after session validation, achievable in one year?
  **A** (@ldgaetano): Similar contracts have already been written; offered direct guidance. kushti asked for concrete bottlenecks before scoping further.

## Open Questions Entering Next Month

- **Scrypto snapshot dependency**: sigmastate-interpreter depends on an unreleased scrypto snapshot (2.1.10), forcing downstream projects to include the sonatype snapshots repo. A release is needed; no timeline confirmed as of month end.
- **ErgoTree pretty-printing**: PRs #812 and #831 for human-readable ErgoTree AST output remain incomplete with no assigned owner.
- **Paideia context variable serialization ordering**: Root cause of the discrepancy between Scala and sigma-rust implementations not identified; the pre-serialized bytes workaround for Nautilus is unconfirmed.
- **Multi-input Ledger signature validity**: Whether reusing a single Ledger-generated signature across multiple inputs with identical scripts is cryptographically valid remains unresolved.
- **Sub-blocks DevNet timeline**: "Within weeks" stated repeatedly but no confirmed release date.
- **EIP-4 token issuance strictness**: Proposal to require all new tokens in a single EIP-4 output for indexing consistency was acknowledged as a good idea but not formally decided.
- **Java/Scala serializer interop for Satergo**: No idiomatic Java syntax found for calling Scala module-based serializers (e.g., `ContextExtension.serializer$.MODULE$.serialize()`).
- **ErgoHack IX anonymous voting via stealth addresses**: kushti suggested possible integration into Paideia DAO; design details remain TBD with no prototype started.
- **L2 wallet concept (ErgoHack IX Idea #9)**: Dual-currency (ERG + stablecoin) Lightning Network-style wallet outlined by kushti; no prototype in progress.
- **Rosen Bridge volunteer testing**: 30+ volunteers requested; recruitment status and testbed completion unconfirmed at month end.
- **Sigma.gallery**: Platform non-functional; repair status unknown.
- **Java 7 / Scala 2.11 legacy dependency deprecation**: kushti asked for a deprecation plan; no response recorded.
- **Keystone hardware wallet integration**: Status requested by kushti in week 39; no update received.
