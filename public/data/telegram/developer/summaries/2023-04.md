# Ergo Developer Chat — Monthly Summary
**Month:** 2023-04 | **Chat:** Ergo Developer

---

## Executive Summary

April 2023 developer activity centred on practical tooling and infrastructure: NFT minting workflows, AppKit integration issues, and node/indexing behaviour. A secondary thread of discussion addressed cross-chain mechanics (Rosen Bridge, Cardano token model comparisons) and mempool/fee market behaviour, with the Ergo protocol reference client v5.0.9 shipped mid-month as the headline release.

---

## Key Topics This Month

### NFT Minting Tooling
Multiple developers worked through NFT bulk minting on Ergo during the month. Dan Friedman explored both Scala and JavaScript approaches, ultimately using the mgPai bulk minter tool. Practical issues encountered included a spurious Base58 encoding exception (resolved by rebuilding the JAR — a compilation/dependency artefact), mnemonic parsing (spaces are the delimiter), and the importance of validating a simple ERG transfer before attempting NFT minting. Luivatra advised treating NFT minting as a "special transaction type" and confirming basic p2pk→p2pk transactions work first.

### AppKit Integration Issues
Two distinct AppKit issues surfaced this month. First, the Satergo developer (Aberg) identified that AppKit's Retrofit/OkHttp3 integration keeps non-daemon HTTP2 threads alive for 5+ minutes post-window-close, blocking clean application shutdown; MrStahlfelge recommended instantiating the REST API client with a custom OkHttp instance and pointed to the Ergo Wallet app source as the reference implementation. Second, `NotEnoughTokensException` was observed returning an empty `tokenBalances` map despite apparently correct usage — suspected causes (zero balance, testnet context, AppKit version regression) remained unresolved at month end.

### Node Infrastructure, Indexing & v5.0.9 Release
The Ergo protocol reference client v5.0.9 was released by kushti on 2023-04-19. Upgrading triggers automatic re-indexing of local node state. Discussions also touched on making indexing status visible on ergonodes.net, node daemon activation, and Luivatra's indexed-node-explorer UI project. Aberg proposed that ergonodes.net's `/list` page should surface whether nodes are indexed — noted as a feature request with no immediate implementation commitment.

### Mempool Behaviour & Fee Markets
A productive exchange clarified Ergo's mempool mechanics: when the mempool reaches capacity (~1,000 TXs), incoming transactions with fees higher than the lowest-fee existing transaction can replace it. Transactions that remain below the standard fee threshold are not permanently stuck — they time out and flush after approximately 10–40 blocks without being picked up by miners. Node operators can also raise the minimum propagation fee rate to prevent low-fee transactions from entering their mempool at all. A fee estimation API exists in the node but was noted as untested and unused by applications in practice.

### Rosen Bridge & Cross-Chain Mechanics
Rosen Bridge was confirmed as the fully decentralised bridge mechanism for Ergo. A related discussion compared Cardano's token transfer model (every token transfer requires ADA attached to the receiving address) with Ergo's fee structure — Luivatra explained that the 2 ERG minimum fee on bridge contracts mirrors this constraint: without it, attackers could flood the pool with tiny outputs, and the 1% additional fee rewards watchers and GARD DAO.

### ErgoScript Type System & ErgoPayments URI Parsing
A focused technical exchange clarified that type code 12 in ErgoScript represents `Coll`, not `Coll<Coll<Byte>>` — the `Coll` naming was confirmed as appropriate for collection types in the compiler. Separately, Aberg flagged that the `#P2PK_ADDRESS#` placeholder in ErgoPayments URIs creates invalid URIs because `#` is interpreted as a fragment delimiter by standard URI parsers (e.g. `java.net.URI`); MrStahlfelge confirmed that percent-encoding (`%23`) is the correct resolution.

## Technical Q&A Worth Preserving

- **Q** (@Dan Friedman, 2023-04-14): What is the best stack for minting NFTs through code? No luck with Scala or JS.
  **A** (@Luivatra): Have you successfully sent a simple transaction first? Start with a 0.1 ERG p2pk→p2pk transfer before NFT minting — NFT minting is a special transaction type. Reference: https://github.com/mgpai22/ergo-nft-bulk-minter

- **Q** (@Dan Friedman, 2023-04-14): NFT minting threw an exception expecting Base58 encoding for token name — do I need to manually encode strings to Base58?
  **A** (@Dan Friedman, self-resolved): Rebuilt the JAR; the issue was a compilation/dependency problem, not a manual encoding requirement.

- **Q** (@Aberg, 2023-04-16): AppKit uses Retrofit/OkHttp3. The OkHttp HTTP2 thread doesn't close after the window closes. How do I access AppKit's OkHttp instance to close it or make it a daemon thread?
  **A** (@MrStahlfelge): Instantiate the REST API client with a custom OkHttp instance — this is recommended practice anyway to avoid multiple thread pools. See the Ergo Wallet app source code for an implementation example. (OkHttp issue reference: https://github.com/square/okhttp/issues/4029)

- **Q** (@Aberg, 2023-04-16): Why is the `NotEnoughTokensException` `tokenBalances` map empty?
  **A** (@Glasgow): The exception is designed to report missing tokens, not found tokens. (Aberg noted the usage pattern was correct but the map remained empty — root cause unresolved; suspected: zero balance, testnet context, or AppKit regression.)

- **Q** (@Flying Pig™, 2023-04-19): What happens if the mempool is full (~1,000 TXs)? Does it reject new TXs or replace the oldest?
  **A** (@kushti): Nodes have fee markets. A new TX with a higher fee than the lowest-fee TX in the mempool can replace it. TXs below the standard fee threshold eventually time out and flush after ~10–40 blocks without being picked up.

- **Q** (@Aberg, 2023-04-30): The ErgoPayments URI spec uses `#P2PK_ADDRESS#` as a placeholder — the `#` makes the URI invalid per `java.net.URI`. How should parsing handle this?
  **A** (@MrStahlfelge): Use percent-encoding — replace `#` with `%23`. Aberg confirmed `%23` is the correct approach.

- **Q** (@Luivatra, 2023-04-30): Does the node return an error immediately if ErgoScript execution cost is too high?
  **A** (@kushti): Yes, the node returns an error immediately if the execution cost exceeds limits.
